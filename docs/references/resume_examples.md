## 3. 경력 기술 (Work Experience) - STAR 기법 적용
경력직 이력서의 핵심은 **'무엇을 했는지(Task)'**보다 **'어떤 성과를 냈는지(Result)'**입니다.

### **[작성 패턴: 문제(Situation) -> 해결(Action) -> 성과(Result)]**

#### **예시 1: 성능 개선**
*   **문제:** 기존 레거시 시스템의 검색 속도 저하로 인한 사용자 이탈 증가.
*   **해결:** RDB 기반 검색을 Elasticsearch로 전환하고, N-gram 인덱싱 및 캐싱 전략 적용.
*   **성과:** 검색 응답 속도 **2초 -> 0.1초(20배) 단축**, 검색 서비스 트래픽 **30% 증가** 수용.

#### **예시 2: 비효율 개선 및 자동화**
*   **문제:** 마케팅팀의 수동 데이터 취합 업무로 인해 일일 3시간 소요.
*   **해결:** 외부 API 연동 및 Spring Batch를 활용한 데이터 수집/정제 파이프라인 구축.
*   **성과:** 리포팅 업무 **100% 자동화**, 마케팅팀 업무 시간 **월 60시간 절감**.

#### **예시 3: 안정성 확보**
*   **문제:** 트래픽 폭주 시 간헐적인 DB 커넥션 풀 고갈 및 장애 발생.
*   **해결:** DB 부하 분산을 위한 Replication 적용 및 Redis 캐싱 레이어 도입, Circuit Breaker 적용.
*   **성과:** 동시 접속자 **10만 명** 상황에서도 **무중단 서비스** 운영 달성.

## 4. 프로젝트 기술서 (Project Description) 구조
이력서와 별도로 제출하는 경력기술서에는 프로젝트별 상세 내용을 기술합니다.

**[프로젝트명] 인플루언서 데이터 플랫폼 고도화**
*   **기간:** 2022.02 - 2023.05
*   **역할:** 백엔드 리드 (기여도 100%)
*   **사용 기술:** Java 17, Spring Boot, Elasticsearch, Redis, Docker
*   **주요 내용:**
    *   **대용량 데이터 처리:** 200만 건 이상의 인플루언서 데이터를 수집/분석하는 파이프라인 구축.
    *   **병렬 처리 최적화:** ThreadPoolTaskExecutor를 활용한 수집 속도 5배 개선.
    *   **안정성 확보:** 외부 API Rate Limit 대응을 위한 Backoff 정책 및 재시도 로직 구현.
*   **핵심 성과:**
    *   데이터 규모 **20배 확대 (10만 -> 200만)**
    *   데이터 수집 속도 **5배 단축**
