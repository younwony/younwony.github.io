### 📌 Situation (배경 및 문제점)

**비즈니스 배경**
- **비즈니스 니즈:** 매출 비중 1위 채널 '네이버 쇼핑'에서 최저가 진입이 필수적이나, 수동 대응으로는 경쟁 속도 미달
- **기술적 제약:** 네이버 API 일일 호출 한도(25,000건) 및 속도 제한으로 전체 상품 실시간 대응 불가

### 🎯 Task (과제/목표)

- API 제약 내 매출 기여도 높은 핵심 상품 선별 및 최저가 방어 알고리즘 구현
- 마진율 4% 보장 범위 내 가격 조정 안전장치 마련
- 자동 + 수동 하이브리드 운영 구조 설계

### ⚡ Action (해결 방법)

**1. 비즈니스 규칙 설계 리딩**
- 마케팅/MD팀과 협의하여 가격 추종 범위(4%), 최소 마진 기준, 예외 상품군 정의
- API Quota 제약을 비즈니스 우선순위로 해결하는 방안 제안 및 합의 도출

**2. Dynamic Pricing 엔진 개발**
- 모델번호 매핑 기반 경쟁사 최저가 수집 후, [자사가 vs 최저가 차이 < 4%] & [최소 마진 확보] 조건 충족 시 할인율 자동 적용

**3. API Quota 최적화**
- 매출 기여도/노출 중요도 기반 상품 우선순위 스케줄링
- Spring Batch로 배치 작업 안정성 확보

**4. 하이브리드 운영 구조 설계**
- 자동 처리 + MD팀 수동 처리 통합
- 예외 케이스 Admin 뷰 제공으로 집중 관리 가능한 업무 플로우 확립

**5. 성과 검증 환경 구축**
- 가격 조정 이력 로그화, GA/BigQuery 연동으로 정량 분석 환경 구축

### ✅ Result (성과)

**정량적 성과**
- **매출/트래픽 10% 동반 상승**: 가격 경쟁력 확보로 해당 상품군 매출·트래픽 각각 10% 증가 (데이터 검증 완료)
- **운영 효율 혁신**: 단순 반복 업무 시스템 전담, MD팀은 전략 상품·예외 케이스에만 집중

**협업 성과**
- **커머스 비즈니스 리딩**: 기술적 제약(API Quota)을 비즈니스 규칙(우선순위)과 협업 프로세스로 해결하는 주도적 역할 수행
- **의사결정 체계 확립**: 마케팅/MD팀과의 정기 협의 체계 구축으로 가격 정책 변경 시 신속한 대응 가능

- 쇼핑몰 노출 이미지와 실제 배송 상품 불일치로 '상품 상이' 클레임 다수 발생

### 🎯 Task (과제/목표)

- 이미지 데이터 정합성 100% 확보로 고객 클레임 원천 차단
- 파일 자체 변경 감지 가능한 고성능/저비용 검증 로직 구현
- 수동 검수 프로세스의 시스템 전환으로 운영 리소스 절감

- Header 우선 비교로 네트워크 비용 최소화

**4. 운영 자동화**
- 24시간 무인 이미지 동기화 운영 체계 구축

### ✅ Result (성과)

**정량적 성과**
- **클레임 Zero 달성**: 이미지 불일치 오배송 클레임 완전 제거
- **반품 비용 절감**: 이미지 상이로 인한 반품 처리 비용 제거
- **Zero Ops 달성**: 수동 검수 → 자동화로 운영 리소스 절감

### 💡 Impact (비즈니스 효과)

- CS팀 클레임 처리 업무 부담 제거
- 브랜드 이미지 보호

### 🔧 Tech Decision

> "MD5 해시 + ETag 이중 검증 - URL 동일해도 이미지 내용 변경 감지, Header 우선 비교로 네트워크 비용 80% 절감"

> "S3 직접 업로드 - CDN 캐시 무효화 없이 즉시 반영, 운영 복잡도 제거"
