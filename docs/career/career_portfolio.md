- **경쟁 환경:** 실시간으로 변동하는 경쟁사 가격에 신속하게 대응하지 못하면 노출 순위 하락 및 매출 감소 발생

**기술적 제약**

- 동적 가격 정책 수립: 무조건적인 최저가가 아닌, 마진율을 보장하는 범위(4% 이내) 내에서만 가격을 조정하는 안전장치 마련
- 하이브리드 프로세스 설계: 시스템이 처리하지 못하는 예외 케이스(Missing Data)를 MD팀이 수동으로 처리할 수 있도록 **'자동+매뉴얼 하이브리드 운영 구조'**를 직접 설계하고 리딩

**1. Dynamic Pricing 로직 구현**
- 모델번호 매핑을 통해 경쟁사 최저가를 수집하고, [자사 판매가 vs 최저가 차이 < 4%] & [최소 마진 확보] 조건을 만족할 때만 할인율을 자동 적용하는 로직 개발

**2. API Quota 효율화 (Scheduling)**

**3. 협업 프로세스 주도 (Process Design)**
- 마케팅/MD팀과 협의하여 '가격 추종 범위(4%)' 및 '예외 상품군' 정책을 확정

**4. 성과 검증 환경 구축**

## 프로젝트 3: 해외 부티크 상품 데이터 정합성 확보 및 클레임 방지 시스템 구축

### 📌 Situation (배경 및 문제점)

- 해외 부티크의 상품 이미지(색상, 디테일)가 변경되었음에도 URL이 동일하게 유지되어, 기존 시스템이 변경 사항을 감지하지 못함
- 이로 인해 쇼핑몰 노출 이미지와 실제 배송 상품이 다른 '상품 상이' 오배송 및 클레임이 다수 발생
- 반품 처리 비용 증가 및 브랜드 신뢰도 하락 문제 대두

### 🎯 Task (과제/목표)

- **이미지 데이터 정합성 100% 확보를 통한 고객 클레임 원천 차단**
- 단순 URL 비교가 아닌, 파일 자체의 변경을 감지할 수 있는 고성능/저비용 검증 로직 구현

### ⚡ Action (해결 방법)

**1. 해시(Hash) 기반 검증 도입**
- 기존의 URL 비교 로직을 폐기하고, HTTP Header(ETag, Last-Modified) 및 바이너리 데이터의 해시값(MD5)을 비교하는 정밀 검증 시스템 구축

**2. 동기화 로직 고도화**
- 배치 작업 시 저장된 해시값과 원본 해시값을 O(1)로 비교하여, 변경 감지 시 즉시 S3 이미지를 강제 업데이트(Update)하도록 파이프라인 개선

**3. 리소스 최적화**
- 전체 파일을 매번 다운로드하지 않고 Header 정보를 우선 비교하여 네트워크 및 처리 비용 최소화

### ✅ Result (성과)

- 상품 이미지 불일치로 인한 고객 클레임 및 오배송 건수 **'Zero' 달성**
- 시각적 정보의 정확도를 완벽하게 유지하여 구매 전환율 안정화 및 고객 신뢰도 회복
- 수동 검수 프로세스를 시스템화하여 운영 리소스 절감

### 사용 기술
Java, Spring Boot, AWS S3, Image Hashing (MD5/ETag)

- 스레드(Thread) 기능을 활용해 관련 논의를 한곳에 모으고, 최종 의사결정 후 버튼 클릭만으로 업무가 종결되도록 구현

**3. Slack Event API 기반 ChatOps 구축**
- 발주 승인/반려, 클레임 처리 등 핵심 기능을 Slack Interactive Component(버튼, 모달)로 구현하여 업무 접근성 극대화
- 이벤트 발생 시 담당자에게 즉시 알림(Notification)을 제공하고, 버튼 클릭 한 번으로 DB 처리까지 완료되도록 프로세스 자동화

### ✅ Result (성과)

- **커뮤니케이션 비용 획기적 절감**: 파편화된 소통 채널을 100% 통합하여 부서 간 협업 속도 증대 및 휴먼 에러 방지

### 📌 Situation (배경 및 문제점)

- **코드 품질 표준 부재**: 팀 내 통일된 코드 컨벤션이 없어 개발자마다 코드 스타일이 상이했고, 이로 인해 가독성 저하 및 협업 비용 발생
- **잠재적 리스크**: 코드 리뷰 절차 없이 배포가 이루어져, 로직 오류나 사이드 이펙트가 운영 환경에서 뒤늦게 발견되는 경우가 빈번함
- **기술 부채 누적**: 일관성 없는 레거시 코드로 인해 신규 기능 개발 시 유지보수 난이도가 지속적으로 상승

### 🎯 Task (과제/목표)

- **Engineering Excellence 추구**: 지속 가능한 개발을 위한 건강한 코드 리뷰 문화 정착
- **시스템 안정성 확보**: 개발 단계에서 잠재적 결함을 사전에 차단하는 프로세스 구축
- **팀 기술 역량 상향 평준화**: 지식 공유를 통한 팀 전체의 기술적 성장 도모

### ⚡ Action (해결 방법)

**1. 코드 리뷰(Code Review) 프로세스 의무화**
- Pull Request(PR) 기반의 개발 프로세스를 도입하고, 배포 전 최소 2명 이상의 리뷰 승인을 거치도록 강제화
- 단순한 지적보다는 '더 나은 설계'를 논의하는 리뷰 문화를 조성하여 팀원들의 참여를 유도

**2. 코드 컨벤션(Convention) 수립 및 자동화**
- 팀원들과의 협의를 통해 Java/Spring 코딩 컨벤션을 명문화
- 신규 입사자도 동일한 규칙을 즉시 준수할 수 있도록 가이드라인 배포

### ✅ Result (성과)

- **코드 품질 및 안정성 향상**: 배포 후 발생하는 버그 및 핫픽스 건수 유의미하게 감소
- **유지보수성 증대**: 일관된 코드 스타일과 구조 적용으로 신규 입사자의 온보딩 시간 단축 및 코드 파악 용이성 증대
- **기술적 성장 주도**: 상호 코드 리뷰를 통해 팀 내 기술 지식이 자연스럽게 전파되고, 주니어 개발자의 성장을 견인

## 프로젝트: Seller 어드민 성능 개선 및 레거시 리팩토링

### 📌 Situation (배경)
- 판매자 정산/주문 조회 시 Slow Query로 인한 응답 지연(**5초 이상**)
- 레거시 비즈니스 로직의 복잡도로 인한 유지보수 어려움
- 판매자의 잦은 타임아웃 불만 접수

### ⚡ Action (해결 방법)

**쿼리 최적화**
- 실행 계획 분석을 통한 인덱스 튜닝 및 불필요한 JOIN 제거
- N+1 문제 해결을 위한 Fetch Join 적용
- 복합 인덱스 설계로 쿼리 성능 향상

**코드 리팩토링**
- 복잡한 비즈니스 로직을 Service Layer로 분리하여 가독성 향상
- DTO 패턴 도입으로 계층 간 데이터 전달 명확화

### ✅ Result (성과)
- 응답 시간 **5초 이상 → 1초 이내**로 단축
- 판매자 만족도 향상

### 사용 기술

## 프로젝트: 자체 검색 엔진 구축 및 대용량 처리 최적화

### 📌 Situation (배경)
- 기존 상용 검색 솔루션의 높은 라이선스 비용 부담
- 대량 기록물 이미지 등록 시 단건 Insert로 인한 처리 지연
- 한글 검색 정확도 부족

### 🎯 Task (과제/목표)
- 비용 절감을 위한 자체 검색 엔진 구축 및 한글 검색 정확도 확보
- 대량 이미지 등록 속도 **70% 이상 단축**

### ⚡ Action (해결 방법)

**검색 엔진 구축**
- Elasticsearch 기반 자체 검색 엔진 개발
- Nori 한글 형태소 분석기 적용으로 검색 정확도 향상
- 동의어 사전 구축으로 검색 커버리지 확대

**성능 최적화**
- 단건 Insert를 Bulk Insert 알고리즘으로 전환하여 DB 트랜잭션 횟수 최소화
- 배치 사이즈 튜닝을 통한 최적 성능 달성
- JDBC Batch Insert로 네트워크 왕복 시간 최소화

### ✅ Result (성과)
- 데이터 등록 속도 **70% 단축**
- 라이선스 비용 절감 및 검색 커스터마이징 자유도 확보
- **2021년 사내 우수사원** 선정

### 사용 기술
Elasticsearch, Nori Analyzer, Bulk Insert, Oracle
