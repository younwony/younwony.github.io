# 윤원희 - 면접 스크립트 (Interview Script)

> **용도**: 면접 시 프로젝트 설명용 서술형 스크립트
> **특징**: 의사결정 과정과 기술 선택 이유를 자연스럽게 설명할 수 있는 서술 형식

## 프로젝트 1: 인플루언서 데이터 플랫폼 고도화

**기간**: 2022.02 - 현재 | **역할**: 백엔드 아키텍처 설계 주도

> 200만 인플루언서 데이터 자동 수집 파이프라인 구축으로 마케팅 커버리지 20배 확대 및 검색 성능 100배 개선

---

마케팅팀이 틱톡/인스타그램을 직접 탐색하며 엑셀로 수기 입력하던 인플루언서 데이터를 자동화하는 과제였습니다. 일 수집량이 수십 명 수준에 불과해 대형 캠페인 진행이 불가능했고, 마케팅팀 리소스의 60% 이상이 데이터 수집과 정리에 소모되고 있었습니다. 또한 200만 건 이상 복합 조건 검색 시 RDB에서 평균 10초 이상 타임아웃이 발생하는 상황이었습니다.

동적 필터 100가지 조합에서 RDB 복합 인덱스로는 한계가 있었습니다. EXPLAIN 분석 결과 type=ALL 풀스캔이 발생했고, Query Caching은 조건 조합의 다양성으로 Hit율 5% 미만이 예상되어 기각했습니다. Materialized View도 검토했으나 5분 이내 실시간 반영 요구사항에 부적합했습니다. 결국 **Elasticsearch 역색인 방식**을 선택했고, 동기화 지연 우려는 Near Real-time Sync(<1분) 파이프라인 구축으로 해소했습니다.

수집 아키텍처는 단일 API 의존 시 Rate Limit으로 일 수집량 한계가 있어, TikTok API + Ensemble API + EchoTik API를 조합한 멀티소스 구조로 설계했습니다. 데이터 중복 문제는 **Redis Distributed Lock**으로 멀티 인스턴스 중복 수집을 방지했습니다. 마케팅팀 주간 리포트 작성에 반나절이 소요되던 문제는 Google Sheets API 기반 자동 리포팅으로 해결했습니다.

| 지표 | Before | After | 개선율 |
|------|--------|-------|--------|
| 일 수집량 | 수십 명 (수작업) | 5,000명 (자동) | **100% 자동화** |
| 데이터 풀 | 10만 명 | 200만 명 | **20배 확대** |
| 검색 응답 | 10초 | 0.1초 | **100배 개선** |
| 리포팅 업무 | 반나절/주 | 자동화 | **Zero Ops** |

결과적으로 대형 캠페인(1,000명+ 규모) 진행이 가능한 마케팅 커버리지를 확보했고, 마케팅팀은 리포팅 업무에서 벗어나 캠페인 기획과 분석 등 고부가가치 업무에 집중할 수 있게 되었습니다. 이 수집 아키텍처 패턴은 타 데이터 수집 프로젝트(상품/가격)의 레퍼런스로 채택되었으며, 정기 소통 체계(주간 싱크) 구축으로 요구사항 반영 속도가 2주에서 3일로 단축되었습니다.

## 프로젝트 2: 네이버 쇼핑 Dynamic Pricing 시스템

**기간**: 2023.01 - 2023.06 | **역할**: 가격 정책 로직 아키텍처 설계 및 운영팀 협의 리드

> API 제약 내 최저가 방어 알고리즘 구현으로 대상 상품군 매출 10% 상승 달성

---

매출 비중 1위 채널인 네이버 쇼핑에서 최저가 진입이 필수였으나, 수동 대응으로는 경쟁사 가격 변동 속도를 따라잡지 못해 노출 순위가 하락하고 있었습니다. 가격 조정 기준도 불명확해 MD팀 의사결정 일관성이 부족했고 마진율 관리에 어려움을 겪고 있었습니다. 네이버 API 일일 호출 한도가 25,000건으로 제한되어 전체 상품(5만+) 실시간 대응은 물리적으로 불가능한 상황이었습니다.

API 호출 한도 내에서 효율을 극대화하기 위해 전체 상품 균등 배분 방식 대신 **매출 기여도 상위 20% 상품 집중 모니터링** 전략을 선택했습니다. 롱테일 상품 대응 지연은 MD팀 수동 모니터링 병행 체계로 보완했습니다. 무제한 가격 추종 시 마진율 침식 위험이 있어, 자사가와 최저가 차이 4% 미만이고 최소 마진이 확보되는 조건에서만 자동 적용하는 **룰 엔진**을 설계했습니다. 4% 초과 시에는 MD팀 수동 승인 프로세스를 거치도록 했습니다.

| 지표 | Before | After | 개선율 |
|------|--------|-------|--------|
| 대상 상품 매출 | 기준 | +10% | **10% 상승** |
| MD팀 가격 조정 업무 | 일 2시간 | 예외 케이스만 | **80% 감소** |

가격 경쟁력 확보로 네이버 쇼핑 노출 순위가 상승하며 유입 트래픽이 증가했고, MD팀은 단순 가격 조정 업무에서 벗어나 상품 기획과 프로모션 전략 등 고부가가치 업무에 집중할 수 있게 되었습니다.

## 프로젝트 3: 상품 이미지 정합성 확보 시스템

**기간**: 2023.07 - 2023.09 | **역할**: 해시 기반 검증 시스템 아키텍처 설계 및 구현 주도

> URL 동일 이미지 변경 감지 시스템 구축으로 이미지 불일치 클레임 Zero 달성

---

해외 부티크에서 상품 이미지를 변경해도 URL이 그대로 유지되는 문제가 있었습니다. 기존 시스템은 URL 기준으로 변경 여부를 판단했기 때문에 이미지 교체를 감지하지 못했고, 쇼핑몰 노출 이미지와 실제 배송 상품이 달라 '상품 상이' 클레임이 월 20건 이상 발생했습니다.

이미지 픽셀 비교 방식은 연산 비용이 과다했습니다. 이미지당 100ms 이상 소요되어 10만 건 처리 시 3시간 이상 걸릴 것으로 예상되었습니다. 대신 **SHA256 해시 검증** 방식을 설계했습니다. 이미지 파일의 첫 0~5KB 영역(메타데이터 포함)만 확인하여 변경 여부를 빠르게 판단하고, 해시값 O(1) 비교로 변경 감지 시 S3에 즉시 업데이트하는 방식을 적용했습니다.

| 지표 | Before | After | 개선율 |
|------|--------|-------|--------|
| 이미지 불일치 클레임 | 월 20건+ | 0건 | **Zero 달성** |
| 검수 업무 | 수동 샘플링 | 전수 자동화 | **Zero Ops** |

## 프로젝트 4: ChatOps 기반 운영 프로세스 자동화

**기간**: 2024.01 - 2024.03 | **역할**: Pain Point 발굴 주도 (인터뷰 10회) 및 ChatOps 아키텍처 설계

> 파편화된 운영 요청 채널을 Slack으로 통합하여 운영팀 수동 업무 40% → 0% 완전 자동화

---

발주와 클레임 처리 요청이 이메일, 개인 메신저, 구두 요청으로 분산되어 있었습니다. 정보 누락과 히스토리 추적 불가로 동일 이슈가 재발했고, 운영팀 업무 시간의 약 40%가 단순 반복 작업에 소모되고 있었습니다.

별도 앱 개발 시 채택률이 저조할 것으로 예상했습니다. 과거 유사한 시도가 실패한 사례가 있었고, 자체 웹 어드민 확장도 접속 허들로 사용률이 저조할 것으로 보였습니다. 구성원이 가장 익숙한 **Slack 기반 ChatOps**를 선택해 진입 장벽을 제거하고 채택률을 극대화했습니다. 버튼과 모달 기반 **Interactive Component**를 설계해 발주 승인/반려, 클레임 처리 등을 버튼 클릭만으로 처리할 수 있도록 구현했습니다.

| 지표 | Before | After | 개선율 |
|------|--------|-------|--------|
| 운영팀 수동 업무 | 40% | 0% | **Zero Ops** |
| 요청 채널 | 5개 분산 | Slack 통합 | **100% 통합** |

## 프로젝트 5: 자체 검색 엔진 구축 (한국문헌정보기술)

**기간**: 2018.06 - 2021.08 | **역할**: 검색 엔진 아키텍처 설계 및 대용량 처리 최적화 주도

> Elasticsearch 기반 자체 검색 엔진 구축으로 연간 라이선스 비용 수천만원 절감 및 데이터 등록 속도 70% 단축

---

상용 검색 솔루션의 연간 라이선스 비용이 수천만원에 달했고, 대량 기록물 이미지를 등록할 때 단건 Insert 방식으로 처리하다 보니 일 8시간 이상이 소요되었습니다.

오픈소스로 라이선스 비용을 Zero로 만들면서 한글 형태소 분석 품질도 우수한 **Elasticsearch + Nori 한글 형태소 분석기**를 선택했습니다. Solr도 검토했으나 한글 분석기 성숙도와 커뮤니티 활성도에서 ES가 우위였습니다. 대량 데이터 등록은 **Bulk Insert**로 전환해 트랜잭션 횟수를 최소화한 결과 등록 속도가 70% 단축되었습니다.

| 지표 | Before | After | 개선율 |
|------|--------|-------|--------|
| 라이선스 비용 | 수천만원/년 | 0원 | **100% 절감** |
| 데이터 등록 속도 | 8시간 | 2.5시간 | **70% 단축** |

이 성과로 **2021년 사내 우수사원**에 선정되었습니다.
