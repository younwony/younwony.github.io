## 경력 내용

```
< Seeding - 인플루언서 마케팅 솔루션 >

[인플루언서 데이터 플랫폼 고도화 - 200만 데이터 규모]
마케팅팀이 틱톡/인스타그램을 수작업으로 탐색하여 엑셀에 수기 입력, 일 수집량 수십 명 수준으로 대형 캠페인 진행 불가
→ TikTok API, Ensemble API, EchoTik API 및 Web Scraper를 조합한 멀티소스 데이터 파이프라인 설계, 병렬 처리 구조로 일 평균 5,000명 자동 수집
→ Elasticsearch 도입으로 복합 조건 검색 성능 100배 개선 (10초 → 0.1초)
→ Redis Distributed Lock으로 멀티 인스턴스 환경에서 API Rate Limit 준수 및 중복 수집 방지
→ 데이터 풀 10만 → 200만명(20배) 확대, 리포팅 수작업 완전 제거(Zero Ops)

< 구하다 - 명품 커머스 >

[네이버 쇼핑 Dynamic Pricing 시스템]
매출 비중 1위 채널 '네이버 쇼핑'에서 최저가 진입 필수, 수동 대응으로는 경쟁 속도 미달
→ API Quota(일 25,000건) 제약 하 매출 기여도 기반 우선순위 스케줄링 설계
→ 마케팅/MD팀과 협의하여 가격 추종 범위(4%), 최소 마진 기준, 예외 상품군 정의
→ Spring Batch로 배치 작업 안정성 확보, 자동+수동 하이브리드 운영 구조 설계
→ 매출·트래픽 10% 동반 상승 달성, MD팀 전략적 의사결정에 집중

[메인 페이지 큐레이션 시스템 구축]
정적 메인 화면 구조로 상품 노출이 기획전 단위로만 가능, MD팀 자율성 제한 및 UI 다양성 부족
→ Enum 기반으로 전시 위치, 아이템 타입, 뷰 타입, 큐레이션 타입 정의하여 확장 용이한 구조 설계
→ Strategy 패턴 적용으로 아이템 타입별 렌더링 로직 분리, OCP 원칙 준수
→ 인메모리 캐시(3분 TTL) 적용으로 외부 의존성 없이 API 응답 속도 최적화
→ MD팀 자율 운영 달성, 아이템 × 뷰 타입 조합으로 다양한 레이아웃 지원

[상품 이미지 정합성 확보 및 클레임 방지]
해외 부티크 상품 이미지 변경 시 URL 동일하게 유지되어 변경 감지 불가, '상품 상이' 클레임 다수 발생
→ SHA256 해시 기반 이미지 변경 정밀 감지 시스템 구축
→ 이미지 파일 첫 0~5KB 영역(메타데이터)만 확인하여 변경 여부 빠르게 판단
→ 클레임 Zero 달성, 반품 처리 비용 제거

[ChatOps 기반 운영 프로세스 자동화]
발주·클레임 처리가 이메일, 메신저, 구두 요청으로 분산, 정보 누락 및 히스토리 추적 불가
→ 운영팀 인터뷰로 가장 큰 병목(어드민 접속 → 데이터 확인 → 메신저 공유 컨텍스트 스위칭) 발굴
→ Slack Event API 기반 Interactive Component로 발주 승인/반려, 클레임 처리 구현
→ 운영팀 반복 업무 40% → 0% 완전 자동화, 채널 100% 통합

[개발 문화 개선 및 시스템 안정성 강화]
통일된 코드 컨벤션과 리뷰 프로세스 부재, 리소스 이슈 감지 체계 부재
→ PR 기반 리뷰 프로세스 도입, 배포 전 최소 2인 승인 체계 확립
→ TDD 기반 개발 도입, 핵심 비즈니스 로직 테스트 커버리지 확보
→ Elastic APM 활용 Slow Query 분석 체계, 디스크/메모리 임계치 Slack 알림 자동화
→ 검색 API TPS 200 → 3,000 달성(1500% 향상), Review API Latency 4초 → 80ms(90% 개선)
→ Claude Code + MCP 활용으로 운영 도구 개발 10시간 → 1시간(90% 단축)
```
