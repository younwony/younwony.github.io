# 자체 검색 엔진 구축 및 대용량 처리 최적화

> 작성일: 2025-12-24
> 소속: (주)한국문헌정보기술 / 솔루션 개발팀
> 기간: 2018.06 - 2021.08

---

## 개요

상용 검색 솔루션 대체를 위한 Elasticsearch 기반 자체 검색 엔진 구축 및 대용량 데이터 처리 최적화 프로젝트.

---

## 배경 및 문제점

### 기존 상황
- 상용 검색 솔루션의 높은 라이선스 비용 부담
- 공공기관 기록물 관리 시스템에 대량 이미지 등록 필요

### 문제점
- **비용 부담**: 상용 검색 솔루션 라이선스 비용
- **성능 저하**: 대량 기록물 이미지 등록 시 단건 Insert로 처리 지연 발생
- **검색 정확도**: 한글 검색 정확도 미흡

### 목표
- 비용 절감을 위한 자체 검색 엔진 구축
- 한글 검색 정확도 확보
- 대량 이미지 등록 속도 70% 이상 단축

---

## 상세 구현

### 아키텍처/설계

```
┌─────────────────────────────────────────────────────────────┐
│                  Application Layer                           │
│                  (Spring Framework)                          │
└─────────────────────────┬───────────────────────────────────┘
                          │
          ┌───────────────┴───────────────┐
          ▼                               ▼
┌─────────────────────┐       ┌─────────────────────┐
│   Elasticsearch     │       │      Oracle         │
│  (Nori Analyzer)    │       │   (Bulk Insert)     │
└─────────────────────┘       └─────────────────────┘
```

### 기술 스택

| 구분 | 기술 |
|------|------|
| 언어 | Java |
| 프레임워크 | Spring Framework |
| 검색엔진 | Elasticsearch |
| 형태소분석 | Nori Analyzer |
| 데이터베이스 | Oracle |

### 구현 내용

#### 1. 검색 엔진 구축
- Elasticsearch 기반 자체 검색 엔진 개발
- Nori 한글 형태소 분석기 적용으로 검색 정확도 향상
- 공공기관 기록물 특성에 맞는 인덱스 설계

#### 2. 성능 최적화
- 단건 Insert → Bulk Insert 전환
- DB 트랜잭션 횟수 최소화
- 배치 사이즈 최적화

### 기술적 의사결정

| 결정 | 선택 | 이유 |
|------|------|------|
| 검색엔진 | Elasticsearch + Nori | 오픈소스로 라이선스 비용 제거, 한글 형태소 분석 정확도 확보 |
| 등록 방식 | Bulk Insert | 트랜잭션 오버헤드 최소화로 대량 등록 성능 개선 |

## 결과 및 성과

### 정량적 성과
- **데이터 등록 속도 70% 단축**
- **라이선스 비용 절감**: 상용 솔루션 대비 비용 제거
- **2021년 사내 우수사원 선정**

### 정성적 효과
- 외부 라이선스 의존성 제거
- 한글 검색 정확도 향상으로 사용자 만족도 개선
- 검색 커스터마이징 자유도 확보
