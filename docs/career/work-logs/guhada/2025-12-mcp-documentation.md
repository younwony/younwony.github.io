# MCP 기반 문서화 자동화

> 작성일: 2025-12-24
> 소속: (주)구하다 / Tech 팀
> 기간: 2025.12

---

## 개요

Claude Code와 Confluence MCP를 연동하여 개발 작업 완료 후 기술 문서를 자동 생성하고, 팀 간 지식 공유를 즉시화한 작업.

---

## 배경 및 문제점

### 기존 상황
- 개발 작업 완료 후 수동으로 Confluence에 문서 작성
- API 개발 시 별도로 API 문서 작성 필요
- 문서 작성에 추가 시간 소요 (작업 대비 30~50% 추가)

### 문제점
- **문서화 지연**: 작업 완료 후 문서 작성까지 시간 gap 발생
- **정보 불일치**: 구두 전달 또는 메신저로 공유 시 정보 누락
- **협업 병목**: 프론트엔드 개발자가 API 스펙 파악에 시간 소요
- **지식 휘발**: 기술적 의사결정 히스토리 누락

### 목표
- 작업 완료 즉시 문서화 가능한 환경 구축
- API 문서 자동 생성으로 프론트엔드 개발자 협업 시간 단축
- 기술적 의사결정 히스토리 축적

### 아키텍처/설계

```
┌─────────────────┐     ┌─────────────────┐     ┌─────────────────┐
│   Claude Code   │────>│   Atlassian     │────>│   Confluence    │

        │

        ▼

│  - 설계 의도    │

└─────────────────┘
```

### 기술 스택

| 구분 | 기술 |
|------|------|
| AI Agent | Claude Code |
| MCP Server | Atlassian MCP |
| 문서 플랫폼 | Confluence |
| 통합 방식 | Model Context Protocol (MCP) |

### 구현 내용

#### 1. Atlassian MCP 서버 연동

Claude Code 설정에 Atlassian MCP 서버 추가:
- Confluence 페이지 조회/생성/수정 가능
- 스페이스 및 페이지 계층 구조 접근

#### 2. 문서화 워크플로우 구축

**작업 완료 후 문서화 프로세스:**
1. 작업 내용 요약 요청 (Claude Code)
2. 코드 변경사항 분석
3. Confluence 페이지 자동 생성
4. 팀 채널 공유

**API 문서 자동화:**
1. Controller/DTO 코드 분석
2. API 스펙 추출 (엔드포인트, 파라미터, 응답)
3. Confluence API 문서 페이지 생성
4. 프론트엔드 개발자 즉시 확인 가능

#### 3. 팀 공유 체계

- **백엔드 테크 팀**: 기술적 의사결정, 아키텍처 변경 문서
- **프론트엔드 팀**: API 스펙 문서, 연동 가이드
- **전체 팀**: 주요 기능 릴리즈 노트

### 기술적 의사결정

| 결정 | 선택 | 이유 |
|------|------|------|
| 문서 플랫폼 | Confluence | 사내 표준 문서 도구, 팀 접근성 높음 |
| 연동 방식 | MCP | 표준화된 프로토콜, Claude Code 네이티브 지원 |
| 문서 형식 | Markdown → ADF | Confluence 렌더링 최적화 |

## 결과 및 성과

### 정량적 성과
- **문서 작성 시간**: 30분 → 5분 (83% 단축)
- **API 연동 문의**: 주 10건 → 2건 (80% 감소)
- **문서화 커버리지**: 주요 작업 100% 문서화

### 정성적 효과
- 작업 컨텍스트가 문서에 즉시 반영되어 정보 정확도 향상
- 프론트엔드 개발자가 API 문서 즉시 확인 가능
- 기술적 의사결정 히스토리 축적
