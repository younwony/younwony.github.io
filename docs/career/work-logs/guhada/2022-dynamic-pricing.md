# 네이버 쇼핑 Dynamic Pricing 시스템

> 작성일: 2025-12-24
> 소속: (주)구하다 / Tech 팀
> 기간: 2022 - 2023

---

## 개요

네이버 쇼핑 최저가 자동 추종 시스템을 구축하여 가격 경쟁력 확보 및 매출 10% 상승을 달성한 프로젝트.

---

## 배경 및 문제점

### 기존 상황
- 매출 비중 1위 채널 '네이버 쇼핑'에서 최저가 진입이 필수
- 수동으로 경쟁사 가격 확인 후 가격 조정

### 문제점
- **비즈니스 니즈**: 수동 대응으로는 경쟁 속도 미달
- **기술적 제약**: 네이버 API 일일 호출 한도(25,000건) 및 속도 제한으로 전체 상품 실시간 대응 불가

### 목표
- API 제약 내 매출 기여도 높은 핵심 상품 선별 및 최저가 방어 알고리즘 구현
- 마진율 4% 보장 범위 내 가격 조정 안전장치 마련
- 자동 + 수동 하이브리드 운영 구조 설계

---

## 상세 구현

### 아키텍처/설계

```
┌─────────────────────────────────────────────────────────────┐
│                   Priority Scheduler                         │
│            (매출 기여도 기반 상품 우선순위)                    │
└─────────────────────────┬───────────────────────────────────┘
                          │
                          ▼
┌─────────────────────────────────────────────────────────────┐
│                 Naver Shopping API                           │
│                (일일 25,000건 제한)                          │
└─────────────────────────┬───────────────────────────────────┘
                          │
                          ▼
┌─────────────────────────────────────────────────────────────┐
│                Dynamic Pricing Engine                        │
│    ┌─────────────────────────────────────────────────┐      │
│    │ 조건: [자사가 vs 최저가 차이 < 4%]               │      │
│    │       AND [최소 마진 확보]                       │      │
│    │ → 할인율 자동 적용                               │      │
│    └─────────────────────────────────────────────────┘      │
└─────────────────────────┬───────────────────────────────────┘
                          │
              ┌───────────┴───────────┐
              ▼                       ▼
┌─────────────────────┐   ┌─────────────────────┐
│    자동 처리        │   │    수동 처리         │
│  (조건 충족 상품)   │   │  (예외 케이스)       │
└─────────────────────┘   └─────────────────────┘
                          │
                          ▼
┌─────────────────────────────────────────────────────────────┐
│              Analytics (GA4 + BigQuery)                      │
└─────────────────────────────────────────────────────────────┘
```

### 기술 스택

| 구분 | 기술 |
|------|------|
| 언어 | Java |
| 프레임워크 | Spring Boot |
| 외부 API | Naver Shopping API |
| 분석 | BigQuery, Google Analytics (GA4) |

### 구현 내용

#### 1. 비즈니스 규칙 설계 리딩
- 마케팅/MD팀과 협의하여 가격 추종 범위(4%), 최소 마진 기준, 예외 상품군 정의
- API Quota 제약을 비즈니스 우선순위로 해결하는 방안 제안 및 합의 도출

#### 2. Dynamic Pricing 엔진 개발
- 모델번호 매핑 기반 경쟁사 최저가 수집
- [자사가 vs 최저가 차이 < 4%] & [최소 마진 확보] 조건 충족 시 할인율 자동 적용

#### 3. API Quota 최적화
- 매출 기여도/노출 중요도 기반 상품 우선순위 스케줄링
- 스케줄링 기반 배치 작업 안정성 확보

#### 4. 하이브리드 운영 구조 설계 (자동화 + MD팀 협업)

**자동 프로세스**:
- 전체 상품 대상 동적 가격 모니터링 및 자동 조정
- 조건 충족 시 할인율 자동 적용

**수동 프로세스 (MD팀 협업 - 핑퐁 방식)**:
- MD팀이 타겟팅한 전략 상품에 대해 데이터 제공
- MD팀이 마진율 계산 → 적정 판매가 산정
- 개발팀이 할인 적용 → 네이버 최저가 달성
- 핑퐁 형태의 협업으로 정밀한 가격 전략 실행
- 예외 케이스 Admin 뷰 제공으로 집중 관리

#### 5. 성과 검증 환경 구축
- 가격 조정 이력 로그화
- GA/BigQuery 연동으로 정량 분석 환경 구축

### 기술적 의사결정

| 결정 | 선택 | 이유 |
|------|------|------|
| 스케줄링 구조 | Spring Boot Scheduler | 정기 배치 실행 및 MD팀 협업 하이브리드 구조로 운영 효율화 |
| API 전략 | 우선순위 스케줄링 | API Quota 25,000건 제약 하 매출 기여도 상위 상품 집중으로 ROI 극대화 |

## 결과 및 성과

### 정량적 성과
- **매출/트래픽 10% 동반 상승**: 가격 경쟁력 확보로 해당 상품군 매출·트래픽 각각 10% 증가

### 정성적 효과
- 단순 반복 업무 시스템 전담, MD팀은 전략 상품·예외 케이스에만 집중
- 가격 경쟁력 확보로 네이버 쇼핑 노출 우위 달성
