# 품절률 감소 및 데이터 분석

> 작성일: 2026-01-04
> 소속: (주)구하다 / Tech 팀
> 기간: 2024

---

## 개요

재구매 유저의 이탈 지점 분석에서 시작하여, 품절 경험이 재구매를 거의 발생시키지 않는다는 인사이트 도출.
DB 기반 품절 연관관계 분석 → 셀러 패널티 정책 수립 → 품절 취소 사유 40% → 20%로 절반 감소 달성.

---

## 배경 및 문제점

### 초기 목표
- **재구매 유저 이탈 지점 파악**: 유저 행동 패턴 분석으로 재구매 과정에서 이탈률이 높은 경험 식별

### 분석 과정
1. Google Analytics BigQuery + Database 연동하여 유저 행동 패턴 분석
2. **핵심 발견**: 품절 경험을 받은 유저가 다음 구매(재구매)로 이어지는 케이스가 거의 없음
3. 품절이 고객 이탈의 핵심 원인임을 데이터로 확인

### 문제점
- **고객 이탈**: 품절 경험 → 재구매 전환율 급감
- **취소 사유 분석**: 전체 취소 중 품절 취소 사유가 40% 차지
- **셀러 관리 한계**: 품절 유발 셀러 식별 및 관리 체계 부재

### 품절 원인 분석 (DB 기반)

| 원인 | 설명 | 대응 가능성 |
|------|------|------------|
| 해외 부티크 데이터 싱크 | 해외 재고 연동 지연 | 구조적 한계 (불가피) |
| **배송 2주 이상 지연** | 셀러 배송 처리 지연 | **패널티 정책 적용** |
| **주문 진행 2주 이상 지연** | 요청→진행 단계 지연 셀러 | **패널티 정책 적용** |

## 상세 구현

### 아키텍처/설계

```
[1단계: 재구매 이탈 분석]
┌─────────────────────────────────────────┐
│         Google Analytics (GA4)           │
│         (유저 행동 데이터)                │
└─────────────────┬───────────────────────┘
                  │
                  ▼
┌─────────────────────────────────────────┐
│              BigQuery                    │
│    (행동 패턴 + 재구매 전환율 분석)       │
└─────────────────┬───────────────────────┘
                  │
                  ▼
        ⚠️ 핵심 발견: 품절 경험 유저 재구매 전환 거의 없음
                  │
                  ▼
[2단계: 품절 원인 분석]
┌─────────────────────────────────────────┐
│              Database                    │
│    (상품/주문/배송/클레임 연관관계)       │
└─────────────────┬───────────────────────┘
                  │
                  ▼
        품절 원인 식별:
        - 해외 부티크 싱크 (불가피)
        - 배송 지연 셀러 (대응 가능)
        - 주문 진행 지연 셀러 (대응 가능)
                  │
                  ▼
[3단계: 정책 수립 및 적용]
┌─────────────────────────────────────────┐
│           셀러 패널티 정책                │
│  ┌─────────────────────────────────┐    │
│  │ • 배송 2주 이상 지연 셀러 패널티  │    │
│  │ • 주문 진행 2주 이상 지연 패널티  │    │
│  │ • 패널티 누적 → 계약 해지        │    │
│  └─────────────────────────────────┘    │
└─────────────────────────────────────────┘
                  │
                  ▼
[4단계: 고객 경험 보상]
┌─────────────────────────────────────────┐
│      품절 경험 고객 포인트 지급           │
│      (이탈 방지 + 재구매 유도)            │
└─────────────────────────────────────────┘
```

### 기술 스택

| 구분 | 기술 |
|------|------|
| 분석 도구 | BigQuery, Google Analytics (GA4) |
| 데이터베이스 | MySQL |
| 시각화 | BigQuery 대시보드 |

### 구현 내용

#### 1. 재구매 이탈 분석 (초기 목표)
- GA4 → BigQuery 연동으로 유저 행동 패턴 분석
- 재구매 유저의 이탈 지점 식별
- **핵심 인사이트**: 품절 경험 유저의 재구매 전환율이 현저히 낮음

#### 2. 품절 연관관계 분석 (원인 파악)
- DB 기반 상품/주문/배송/클레임 연관관계 분석
- 품절 원인 분류:
  - 해외 부티크 데이터 싱크 (구조적 한계)
  - **배송 2주 이상 지연 셀러** (대응 가능)
  - **주문 진행 2주 이상 지연 셀러** (대응 가능)

#### 3. 셀러 패널티 정책 수립 및 적용
- 배송/주문 진행 2주 이상 지연 셀러 대상 패널티 부여
- 패널티 누적 시 계약 해지까지 진행
- 정책을 통한 셀러 품질 관리 체계 구축

#### 4. 고객 경험 보상 시스템
- 품절 경험 고객 대상 일정량 포인트 지급
- 이탈 방지 및 재구매 유도

### 기술적 의사결정

| 결정 | 선택 | 이유 |
|------|------|------|
| 분석 도구 | BigQuery + GA4 | 유저 행동 데이터와 내부 DB 조인 분석 용이 |
| 접근 방식 | 고객 관점 | "재구매 이탈"이라는 고객 경험 문제에서 출발 |
| 해결 방식 | 정책 + 보상 | 원인 제거(셀러 패널티) + 경험 보상(포인트) 병행 |
