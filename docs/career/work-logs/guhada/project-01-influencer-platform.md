# 인플루언서 데이터 플랫폼 고도화 (200만 데이터 규모)

> 작성일: 2025-12-24
> 소속: (주)구하다 / Tech 팀
> 기간: 2022.02 - 진행 중

---

## 개요

마케팅팀의 수작업 인플루언서 탐색/관리를 100% 자동화하고, 200만 건 규모의 데이터 플랫폼을 구축한 프로젝트.

---

## 배경 및 문제점

### 기존 상황
- 마케팅팀이 틱톡/인스타그램을 직접 탐색 후 엑셀에 수기 입력하여 관리
- 일 수집량이 수십 명 수준에 불과

### 문제점
- **수작업 의존**: 인적 리소스 한계 및 데이터 최신성 유지 불가
- **캠페인 확장 한계**: 대형 캠페인 진행 및 다양한 인플루언서 매칭 불가
- **데이터 파편화**: 멀티 플랫폼 지표의 통합 분석 기준 부재
- **검색 성능 저하**: 200만 건 이상 복합 조건 검색 시 RDB 쿼리 타임아웃 (평균 10초 이상)

### 목표
- 수집 프로세스 100% 자동화 및 정밀 타겟팅 구현
- 이미지 데이터 정합성 100% 확보
- 200만 건 규모 실시간 검색 및 분석 기능 제공

## 상세 구현

### 아키텍처/설계

```
┌─────────────────────────────────────────────────────────────┐
│                    Data Collection Layer                     │
├──────────────┬──────────────┬──────────────┬────────────────┤
│  TikTok API  │ Ensemble API │ EchoTik API  │  Web Scraper   │
└──────┬───────┴──────┬───────┴──────┬───────┴────────┬───────┘
       │              │              │                │
       ▼              ▼              ▼                ▼
┌─────────────────────────────────────────────────────────────┐
│              Parallel Processing Engine                      │
│         (Redis Distributed Lock + Retry Logic)               │
└─────────────────────────┬───────────────────────────────────┘
                          │
                          ▼
┌─────────────────────────────────────────────────────────────┐
│                      Storage Layer                           │
├────────────────────────┬────────────────────────────────────┤
│      MySQL (RDB)       │     Elasticsearch (검색)           │
└────────────────────────┴────────────────────────────────────┘
                          │
                          ▼
┌─────────────────────────────────────────────────────────────┐
│                  Reporting Automation                        │
│                  (Google Sheets API)                         │
└─────────────────────────────────────────────────────────────┘
```

### 기술 스택

| 구분 | 기술 |
|------|------|
| 언어 | Java |
| 프레임워크 | Spring Boot |
| 데이터베이스 | MySQL |
| 검색엔진 | Elasticsearch |
| 캐시/락 | Redis (Distributed Lock) |
| 외부 API | TikTok API, Ensemble API, EchoTik API |
| 리포팅 | Google Sheets API |

### 구현 내용

#### 1. 요구사항 분석 및 협업 리딩
- 마케팅팀 정기 미팅으로 Pain Point 파악
- 수집 조건 우선순위 공동 정의
- 기술적 제약사항을 비기술 부서가 이해할 수 있도록 설명

#### 2. 멀티소스 데이터 파이프라인 설계
- TikTok API, Ensemble API, EchoTik API 및 Web Scraper 조합
- 병렬 처리 구조로 일 5,000명 자동 수집 달성

#### 3. 시스템 안정성 확보
- Retry 로직으로 API 차단 방지 및 자동 복구 처리
- Redis Distributed Lock으로 멀티 인스턴스 환경 중복 수집 방지

#### 4. 검색 성능 최적화
- Elasticsearch 도입으로 복합 조건 검색 10초 → 0.1초 (100배 개선)

#### 5. 운영 자동화
- Google Sheets API 기반 리포팅 완전 자동화

### 기술적 의사결정

| 결정 | 선택 | 이유 |
|------|------|------|
| 검색엔진 | Elasticsearch | 200만 건 복합 조건 검색에서 RDB는 10초 이상 타임아웃, 역색인 기반 ES로 0.1초 달성 |
| 동시성 제어 | Redis Distributed Lock | 멀티 인스턴스 환경에서 중복 수집 방지 및 API Rate Limit 준수 |
| 리포팅 | Google Sheets API | 마케팅팀 익숙한 도구, 실시간 업데이트 |

## 결과 및 성과

### 정량적 성과
- **수집 자동화**: 수작업 → 일 5,000명 자동 수집 (100% 자동화)
