# 상품상세 3차 업데이트 (Review API 개선)

> 작성일: 2025-12-31
> 소속: (주)구하다 / Tech 팀
> 기간: 2023.03 - 2023.05

---

## 개요

상품 상세 Review 영역 개선 프로젝트. 객체화 구조 채용과 Slow Query 튜닝을 통해 API Latency를 대폭 개선.

---

## 배경 및 문제점

### 기존 상황
- 상품 상세 Review API 운영 중
- 초기 API 배포 시 Latency가 높게 측정됨

### 문제점
- **높은 Latency**: 100ms ~ 4,000ms로 응답 시간 편차 큼
- **유지보수 어려움**: 코드 구조 개선 필요

### 목표
- Review API Latency 개선
- 유지보수 용이한 코드 구조 구성

---

## 상세 구현

### 기술 스택

| 구분 | 기술 |
|------|------|
| 언어 | Java |
| 프레임워크 | Spring Boot |
| 데이터베이스 | MySQL |

### 구현 내용

#### 1. 객체화 구조 채용
- Code Level에서 객체화 구조 채용
- 유지보수 높은 코드 구조 구성

#### 2. Slow Query 튜닝
- Select Query 분석 및 튜닝
- 인덱스 최적화

### 기술적 의사결정

| 결정 | 선택 | 이유 |
|------|------|------|
| 코드 구조 | 객체화 구조 | 유지보수성 향상 및 확장성 확보 |
| 성능 개선 | Query 튜닝 | Latency 병목의 주요 원인 |

---

## 결과 및 성과

### 정량적 성과
- **Latency 최대 90% 개선**: 100ms ~ 4,000ms → 30ms ~ 80ms
- 평균 Latency 50ms ~ 100ms로 안정화

### 정성적 효과
- 유지보수 용이한 코드 구조 확보
- 사용자 상품 상세 조회 경험 향상
