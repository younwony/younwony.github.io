# Seller 어드민 성능 개선 및 레거시 리팩토링

> 작성일: 2025-12-24
> 소속: (주)인터파크 / 쇼핑 개발팀
> 기간: 2021.09 - 2022.01

---

## 개요

판매자 정산/주문 조회 Slow Query 개선 및 레거시 코드 리팩토링으로 조회 성능을 80% 단축한 프로젝트.

---

## 배경 및 문제점

### 기존 상황
- 판매자 정산/주문 조회 시 Slow Query로 응답 지연 5초 이상 발생
- 레거시 비즈니스 로직 복잡도로 유지보수 어려움

### 문제점
- **성능 저하**: 판매자가 정산/주문 조회 시 5초 이상 대기
- **유지보수 어려움**: 복잡한 레거시 코드로 기능 추가/수정 시 리스크 증가

### 목표
- 판매자 어드민 조회 성능 개선 (5초 → 1초 미만)
- 레거시 코드 가독성 및 유지보수성 향상

---

## 상세 구현

### 기술 스택

| 구분 | 기술 |
|------|------|
| 언어 | Java |
| 프레임워크 | Spring Framework |
| 데이터베이스 | Oracle |

### 구현 내용

#### 1. 쿼리 최적화
- 실행 계획(Execution Plan) 분석
- 인덱스 튜닝 및 불필요한 JOIN 제거
- 서브쿼리 → 조인 변환

#### 2. 코드 리팩토링
- 복잡한 비즈니스 로직을 Service Layer로 분리
- 메소드 단일 책임 원칙 적용
- 가독성 향상을 위한 구조 개선

### 기술적 의사결정

| 결정 | 선택 | 이유 |
|------|------|------|
| 최적화 방식 | 인덱스 튜닝 + 쿼리 재작성 | 기존 테이블 구조 변경 없이 성능 개선 가능 |
| 리팩토링 범위 | Service Layer 분리 | 비즈니스 로직과 데이터 접근 로직 분리로 유지보수성 향상 |

## 결과 및 성과

### 정량적 성과
- **조회 성능 개선**: 5초 → 1초 미만 (80% 단축)
- **코드 가독성 향상**: 유지보수 효율 개선

### 정성적 효과
- 판매자 어드민 사용성 개선
- 신규 기능 추가 시 개발 속도 향상
