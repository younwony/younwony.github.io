## 1. 협력하는 객체들의 공동체

**작성일**: 2023.01.17

### 핵심 개념

> 객체지향의 목표는 **"새로운 세계를 창조"**하는 것

협력에는 **역할**과 **책임**이 있습니다.

### 역할, 책임, 협력

> "적절한 객체에게 적절한 책임을 할당하는 것"이 객체지향 설계의 시작점입니다.

### 협력 속에 사는 객체

객체의 특징:
- **협력적**이면서도 **자율적**
- **상태**와 **행동**을 함께 지님

### 객체지향의 본질

- 시스템을 자율적 객체들의 공동체로 봄
- 메시지 전송을 통해 협력
- 각 객체는 고유한 역할과 책임을 가짐

### 핵심 메시지

> **"객체지향은 객체를 지향하는 것이지 클래스를 지향하는 것이 아니다"**

클래스의 구조보다 **객체의 역할, 책임, 협력**에 집중해야 합니다.

## 2. 이상한 나라의 객체

**작성일**: 2023.01.17

### 객체의 핵심 특징

1. **상태**를 가지며 변경 가능
2. **행동**으로 상태 변경
3. **유일한 식별자** 보유

### 소프트웨어 세계의 객체

객체의 구성 요소:
- **상태** (프로퍼티)
- **행동**
- **식별자**

> "객체의 상태를 변경하는 것은 객체의 자발적인 행동뿐"

협력 관계의 중요성을 강조합니다.

### 설계의 핵심 원칙

> **"행동이 상태를 결정한다"**

| 설계 방식 | 특징 | 문제점 |
|----------|------|--------|
| 상태 중심 설계 | 초보자의 일반적 실수 | 캡슐화 위반, 변경에 취약 |
| **책임-주도 설계(RDD)** | 행동 중심 | 유연하고 확장 가능 |

### 동등성 vs 동일성

| 구분 | 비교 대상 | 비교 방식 |
|------|----------|----------|
| 동등성 (equality) | 값 | 상태로 비교 |
| 동일성 (identity) | 객체 | 식별자로 비교 |

---

## 핵심 정리

### 객체지향의 본질
- 객체는 역할, 책임, 협력의 집합
- 클래스가 아닌 객체를 중심으로 생각
- 메시지를 통한 협력

### 좋은 설계 원칙
- 행동이 상태를 결정
- 책임-주도 설계(RDD) 적용
- 객체의 자율성 보장

### 객체의 특징
- 상태 + 행동 + 식별자
- 협력적이면서 자율적
- 메시지를 통해 소통
