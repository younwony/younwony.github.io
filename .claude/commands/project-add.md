# 새 프로젝트 추가

> `my_career_data.md`에 새 프로젝트를 STAR+I 형식으로 추가합니다.
> **참조 Skill**: `resume-writing` (STAR+I 기법, 정량화 방식, 시니어 톤)

---

## 실행 단계

### 1단계: 프로젝트 정보 수집

사용자에게 확인할 정보:

1. **프로젝트명**: 제목
2. **기간**: 시작일 - 종료일
3. **Situation**: 문제점, 제약사항
4. **Task**: 해결 목표
5. **Action**: 기술적 의사결정 (Why & How)
6. **Result**: 정량적 성과 (숫자)
7. **Impact**: 비즈니스 효과

### 2단계: STAR+I 형식으로 작성

```markdown
### 프로젝트 N: [프로젝트명]

**기간**: YYYY.MM - YYYY.MM

#### Situation (상황)
- 기존 시스템의 문제점

#### Task (과제)
- 해결 목표

#### Action (실행)
- 기술적 의사결정 (**왜 그 기술을 선택했는지**)
- 사용한 기술 및 방법론

#### Result (결과)
- 정량적 성과 (숫자로 표현)

#### Impact (비즈니스 효과)
- 조직적/비즈니스 임팩트
```

### 3단계: 10개 파일 업데이트

`/resume-update` 명령어 실행

### 4단계: 기술 스택 업데이트

새 프로젝트에서 사용한 기술이 기존에 없다면 추가:
- **사용 중인 기술만** 포함
- 배운 것만 나열 금지

### 5단계: 검증 및 커밋

```bash
git diff docs/career/my_career_data.md
git status
```
