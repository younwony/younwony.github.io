# 맞춤형 이력서 생성

## 개요

원본 데이터(`my_career_data.md`)를 기반으로 JD 또는 기업 유형에 맞춰 최적화된 이력서를 생성합니다.

## 입력 유형

### 1. JD 기반 (URL 또는 텍스트)
```
사용자: "이 JD에 맞춰 이력서 만들어줘: [URL 또는 텍스트]"
```

### 2. 기업 유형 기반
```
사용자: "스타트업용 이력서 만들어줘"
사용자: "대기업 지원용으로 이력서 조정해줘"
```

### 3. 도메인 기반
```
사용자: "핀테크 회사 지원용 이력서"
사용자: "커머스 도메인 강조해서"
```

## 실행 단계

### Step 1: 입력 분석

**JD 분석 시 추출 항목:**
- 필수 기술 스택
- 우대 사항
- 주요 업무 내용
- 도메인/산업
- 연차 요구사항

**키워드 매핑:**
| JD 키워드 | my_career_data.md 매칭 |
|----------|----------------------|
| 대용량 트래픽 | 인플루언서 플랫폼 (200만 건) |
| 가격/정산 | Dynamic Pricing, 커머스 도메인 |
| 자동화 | ChatOps, Zero Ops |
| 검색 엔진 | Elasticsearch |
| 배치 처리 | Spring Batch |

### Step 2: 강조점 결정

**기업 유형별 강조 매트릭스:**

| 유형 | 1순위 | 2순위 | 3순위 | 톤 |
|------|-------|-------|-------|-----|
| **스타트업** | 자동화/생산성 | 빠른 실행력 | 풀스택 역량 | 능동적, 도전적 |
| **대기업** | 대규모 처리 | 안정성/품질 | 협업/리더십 | 체계적, 신뢰감 |
| **핀테크** | 데이터 정합성 | 성능 최적화 | 도메인 이해 | 정밀함, 책임감 |
| **커머스** | 도메인 경험 | 매출 성과 | 운영 자동화 | 비즈니스 중심 |
| **SaaS/플랫폼** | API 설계 | 확장성 | 모니터링 | 기술 깊이 |
| **에이전시** | 다양한 프로젝트 | 빠른 적응력 | 커뮤니케이션 | 유연함 |

### Step 3: 콘텐츠 재구성

**Professional Summary 조정:**
- JD 핵심 키워드를 첫 문장에 배치
- 관련 성과를 우선 노출

**프로젝트 순서 재배치:**
- JD와 가장 관련 높은 프로젝트를 상위로
- 관련 없는 프로젝트는 간소화 또는 생략

**기술 스택 강조:**
- JD 필수 기술을 Bold 처리
- JD에 없는 기술은 하위로 이동

### Step 4: 출력

**파일 생성:**
```
docs/career/formats/by-jd/{company}_{date}.md
```

**예시:**
```
docs/career/formats/by-jd/toss_2024-12.md
docs/career/formats/by-jd/coupang_2024-12.md
```

## 프로젝트-키워드 매핑 가이드

### 인플루언서 데이터 플랫폼
**강조 키워드:** 대용량, ETL, 검색엔진, 데이터 파이프라인, Elasticsearch, 200만 건
**적합 JD:** 데이터 엔지니어링, 검색 플랫폼, 대용량 처리

### Dynamic Pricing
**강조 키워드:** 커머스, 가격, 매출, API 연동, 배치 처리, Spring Batch
**적합 JD:** 커머스, 핀테크, 정산, 가격 시스템

### 이미지 정합성
**강조 키워드:** 데이터 정합성, S3, 해싱, 무결성
**적합 JD:** 데이터 품질, 미디어 플랫폼

### ChatOps
**강조 키워드:** 자동화, Slack API, 운영 효율화, Zero Ops
**적합 JD:** DevOps, SRE, 플랫폼 엔지니어링

### AI 활용 개발
**강조 키워드:** AI 코딩, 생산성, 코드 리뷰, 개발 문화
**적합 JD:** 테크 리드, 시니어 개발자, 개발 문화

## 출력 형식

### Markdown
```markdown
# 윤원희 - {회사명} 지원용 이력서

> 이 이력서는 {회사명} JD를 기반으로 최적화되었습니다.
> 생성일: {날짜}

## Professional Summary
[JD 맞춤 요약]

## 핵심 역량 (JD 매칭)
[JD 키워드 기반 재구성]

## 관련 프로젝트
[JD 관련도 순으로 정렬]
```

## 체크리스트

### 분석 단계
- [ ] JD 키워드 추출 완료
- [ ] 기업 유형 파악
- [ ] 도메인 확인

### 매칭 단계
- [ ] 관련 프로젝트 선별
- [ ] 강조 기술 스택 결정
- [ ] 프로젝트 순서 재배치

### 출력 단계
- [ ] Professional Summary 조정
- [ ] 핵심 성과 JD 맞춤 표현
- [ ] 파일 생성 및 저장
- [ ] git add 완료

## 예시

### 입력
```
토스 백엔드 개발자 JD:
- 필수: Java, Spring Boot, 대용량 트래픽 처리
- 우대: 핀테크/결제 경험, Kafka
- 업무: 결제 시스템 개발, 정산 자동화
```

### 출력 (Summary 부분)
```markdown
## Professional Summary

**대용량 트래픽과 정산 시스템 전문 백엔드 개발자**

Java/Spring 기반 7년차 백엔드 개발자로서 **200만 건 규모 데이터 처리**와
**가격/정산 시스템 자동화**를 전문으로 합니다.

**관련 성과:**
- 네이버 쇼핑 Dynamic Pricing으로 **매출 10% 상승** (가격 경쟁력 확보)
- 200만 건 실시간 검색 **100배 성능 개선** (10초 → 0.1초)
- 운영 프로세스 **Zero Ops 달성** (자동화 100%)
```
