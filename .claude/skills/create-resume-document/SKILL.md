# 이력서 문서 작성 (2-3페이지)

## 개요

PDF 출력에 최적화된 **2-3페이지 분량의 이력서**를 작성합니다.
경력기술서(5페이지+)와 달리, 채용 담당자가 빠르게 스크리닝할 수 있도록 핵심 성과 중심으로 작성합니다.

> **주의**: PDF 내보내기는 `/export-pdf` 스킬을 사용하세요.

## 이력서 vs 경력기술서 구분

| 구분 | 이력서 (Resume) | 경력기술서 (Career Portfolio) |
|------|----------------|------------------------------|
| **분량** | 2-3페이지 | 5페이지 이상 |
| **목적** | 빠른 스크리닝 | 상세 역량 검증 |
| **내용** | 핵심 성과 요약 | 프로젝트 STAR+I 상세 |
| **톤** | 간결, 숫자 강조 | 구체적, 서술적 |
| **템플릿** | `resume-2page.html` | `career-portfolio.html` |

## 템플릿 구조

### 페이지 1: 핵심 정보

```
┌─────────────────────────────────────────┐
│ Header: 이름, 직함, 연락처                │
├─────────────────────────────────────────┤
│ Key Metrics: 핵심 성과 4개 (숫자 강조)     │
│ [Zero Ops] [200만] [100x] [+10%]        │
├─────────────────────────────────────────┤
│ Professional Summary (3-4줄)             │
│ - 핵심 역량 + 대표 성과                   │
├─────────────────────────────────────────┤
│ Core Competencies (6개)                  │
├─────────────────────────────────────────┤
│ Technical Skills (2x2 그리드)             │
│ - Backend, Database, Infra, Tools       │
└─────────────────────────────────────────┘
```

### 페이지 2: 경력 & 프로젝트

```
┌─────────────────────────────────────────┐
│ Work Experience                          │
│ ├── 회사 1: 핵심 성과 4-5줄              │
│ ├── 회사 2: 핵심 성과 2-3줄              │
│ └── 회사 3: 핵심 성과 2줄                │
├─────────────────────────────────────────┤
│ Key Projects (카드 형식)                  │
│ - 프로젝트 1: 제목 + 설명 + 기술스택      │
│ - 프로젝트 2                             │
│ - 프로젝트 3                             │
└─────────────────────────────────────────┘
```

### 페이지 3: 추가 프로젝트 & 학력 (선택)

```
┌─────────────────────────────────────────┐
│ Key Projects (continued)                 │
│ - 프로젝트 4                             │
├─────────────────────────────────────────┤
│ Education & Certification               │
├─────────────────────────────────────────┤
│ Awards                                   │
└─────────────────────────────────────────┘
```

## 작성 원칙 (writing-guide 기반)

### 1. 숫자로 말하기

모든 성과는 정량화합니다:

```
❌ "성능 개선함"
✅ "검색 성능 100배 개선 (10초→0.1초)"

❌ "자동화 시스템 구축"
✅ "운영 수동 업무 40%→0% (Zero Ops)"
```

### 2. 시니어 톤 사용

| 피해야 할 표현 | 권장 표현 |
|--------------|----------|
| 담당했습니다 | **달성/개선/안정화했습니다** |
| 구현했습니다 | **설계하고 구현했습니다** |
| 사용했습니다 | **선택·도입했습니다** |

### 3. 핵심만 남기기

- 한 성과는 **1-2줄** 이내
- 경력 설명은 **3-5개 bullet** 이내
- 프로젝트 카드는 **3-4줄** 설명

### 4. 최신 경력 우선

```
회사 1 (현재): 5개 성과 상세
회사 2 (직전): 3개 성과
회사 3 (이전): 2개 성과 간략
```

## 이력서에 포함할 내용

- ✅ 핵심 성과 (숫자 필수)
- ✅ 기술 스택 요약
- ✅ 최근 3개 회사 경력
- ✅ 대표 프로젝트 3-4개
- ✅ 학력/자격증/수상
- ❌ 프로젝트 상세 STAR+I (경력기술서용)
- ❌ 기술 선택 이유 (경력기술서용)
- ❌ 모든 프로젝트 나열

## 실행 단계

### Step 1: 원본 데이터 확인

```bash
# 최신 데이터 확인
cat docs/career/my_career_data.md
```

### Step 2: 템플릿 선택

| 상황 | 템플릿 |
|------|--------|
| 채용 사이트 제출 | `resume-2page.html` (2-3페이지) |
| 이메일 첨부 | `resume-2page.html` (2-3페이지) |
| 상세 제출 | `resume-2page.html` (2-3페이지) |

### Step 3: 이력서 작성

1. 원본 데이터에서 핵심 성과 추출
2. 시니어 톤으로 문장 다듬기
3. 숫자 강조 (뱃지/볼드 처리)
4. 페이지 분량 조절

### Step 4: 레이아웃 검토

PDF 출력 전 반드시 확인:

- [ ] 2-3페이지 분량 준수
- [ ] 각 페이지가 자연스럽게 끝남
- [ ] 섹션이 페이지 경계에서 잘리지 않음
- [ ] 숫자 성과가 눈에 띄게 표시됨

> **PDF 내보내기**: `/export-pdf` 스킬을 사용하세요.

## 성과 작성 예시

### Work Experience 섹션

```html
<ul class="achievements">
    <li>
        <strong>인플루언서 플랫폼:</strong>
        200만 건 파이프라인 구축, 검색 성능

    </li>
    <li>
        <strong>Dynamic Pricing:</strong>
        네이버 쇼핑 가격 자동화
        <span class="badge">매출 10%↑</span>
    </li>
</ul>
```

```html
<div class="project">
    <div class="project-title">인플루언서 데이터 플랫폼 (200만 데이터)</div>

        마케팅팀 수작업 완전 제거.
    </div>
    <div class="project-tech">
        <span class="tech-tag">Spring Boot</span>
        <span class="tech-tag">Elasticsearch</span>
    </div>
</div>
```
