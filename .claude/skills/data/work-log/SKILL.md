# Work Log - 작업 내용 기록 및 이력서 반영

## 개요

특정 작업 완료 후 해당 경험을 **프로젝트별 작업 로그**에 상세 기록하고, 이력서 반영 여부를 판단합니다.

## 디렉토리 구조

```
docs/career/
├── my_career_data.md          # 이력서 원본 (SSOT)
├── resume.md                  # 기본 이력서
├── career_portfolio.md        # 경력기술서
└── work-logs/                 # 작업 로그 (프로젝트별)
    ├── README.md              # 작업 로그 인덱스
    ├── guhada/                # 구하다 프로젝트
    │   ├── 2024-mcp-documentation.md
    │   ├── 2024-chatops-enhancement.md
    │   └── ...
    ├── interpark/             # 인터파크 프로젝트
    │   └── ...
    └── personal/              # 개인 프로젝트/학습
        └── ...
```

## 처리 흐름

```
작업 내용 수집
    ↓
프로젝트별 작업 로그 작성 (상세)
    ↓
이력서 반영 판단
    ↓
┌─────────────────────────────────────────────┐
│ 반영: 이력서 업데이트                         │
│   ├─ 기존 프로젝트 보강 (연관성 있음)          │
│   └─ 새 프로젝트로 추가 (독립적 성과)          │
├─────────────────────────────────────────────┤
│ 미반영: 작업 로그에만 기록 (상세 내용 유지)     │
└─────────────────────────────────────────────┘
```

## 실행 단계

### Step 1: 작업 내용 수집

사용자에게 아래 정보 요청:

```markdown
## 기본 정보
- 작업명:
- 소속 회사/프로젝트:
- 수행 기간: YYYY.MM

## 작업 배경
- 왜 이 작업을 했는가?
- 기존 문제점/Pain Point는?

## 작업 상세
- 무엇을 했는가? (구체적으로)
- 어떤 기술/도구를 사용했는가?
- 기술적 의사결정은?

## 결과
- 정량적 성과 (가능하면)
- 정성적 효과

## 팀/조직 영향
- 다른 팀원/부서에 미친 영향
- 공유/전파 여부
```

### Step 2: 프로젝트별 작업 로그 작성

**파일 위치:** `docs/career/work-logs/{company}/{파일명}.md`

**파일명 규칙:**

| 형식 | 사용 시점 | 예시 |
|------|----------|------|
| `YYYY-MM-작업명.md` | 월이 명확한 경우 | `2025-12-mcp-documentation.md` |
| `YYYY-작업명.md` | 연도만 있는 경우 | `2022-dynamic-pricing.md` |

**템플릿:**
```markdown
# [작업명]

> 작성일: YYYY-MM-DD
> 소속: [회사명] / [팀명]
> 기간: YYYY.MM
```

### Step 3: 이력서 반영 판단

**이력서 반영 기준:**

| 기준 | 설명 |
|------|------|
| **정량적 성과** | 수치로 표현 가능 (%, 배, 시간 등) |
| **기술적 깊이** | 의사결정 과정, 트레이드오프 고려 |
| **비즈니스 임팩트** | 팀/조직 수준의 영향 |
| **스토리텔링** | 면접에서 설명할 수 있는 스토리 |

**작업 로그만 기록:**
- 일상적 운영 업무
- 정량적 성과 불명확
- 개인 학습/실험

### Step 4: 반영 실행

#### Case A: 기존 프로젝트 보강

1. `my_career_data.md` 연관 프로젝트 Action 섹션에 추가
2. `/update-resume` 스킬로 10개 파일 동기화

#### Case B: 새 프로젝트로 추가

1. `/add-project` 스킬 실행

#### Case C: 작업 로그만 기록

작업 로그 파일만 저장 (상세 내용 유지)

### Step 5: README.md 업데이트

`docs/career/work-logs/README.md`에 인덱스 추가:

```markdown
## 구하다

| 날짜 | 작업명 | 이력서 반영 |
|------|--------|------------|
| 2024.12 | MCP 기반 문서화 자동화 | 프로젝트 5 |
| 2024.11 | 로그 조회 스크립트 | - |
```

### Step 6: git add

```bash
git add docs/career/work-logs/
# 이력서 반영 시 추가

git add templates/resume/*.html

```
