# 7년차 백엔드 개발자 이력서 작성 가이드

## 핵심 마인드셋

| Before | After |
|--------|-------|
| 개발자 (Coder) | **전략가** - 기술로 비즈니스 문제를 해결하는 사람 |
| "담당했습니다" | **"달성/개선했습니다"** - 능동적 성과 강조 |
| 모든 경험 나열 | **상위 20% 핵심 프로젝트 집중** |

> **핵심**: "무엇을 했다(What)"가 아닌 **"왜(Why) → 어떻게(How) → 성과(Impact)"**

---

## STAR+I 프레임워크

### 구조

| 요소 | 설명 | 핵심 질문 |
|------|------|----------|
| **S** (Situation) | 배경·문제·제약 | 어떤 어려움이 있었는가? |
| **T** (Task) | 과제/목표 | 해결해야 할 과제는? |
| **A** (Action) | 기술적 의사결정 | **왜 그 기술을 선택했는가?** |
| **R** (Result) | 정량적 성과 | 수치로 증명된 결과는? |
| **I** (Impact) | 비즈니스 효과 | 조직에 미친 영향은? |

### 문장 템플릿

```
"[제약/문제](S) 속에서 [기술적 의사결정](A)으로 [정량적 성과](R) 달성,
[비즈니스/운영 효과](I)"
```

### 예시

> "API Limit 25k 제약(S) 속에서 우선순위 큐 기반 가격조정 로직 설계(A)로
> 핵심 상품 대응률 90% 달성(R), 운영팀 수작업 완전 제거(I)"

## ❌ Bad vs ✅ Good 표현

| 구분 | ❌ 나쁜 표현 | ✅ 좋은 표현 |
|------|------------|------------|
| **데이터 수집** | 데이터 크롤링 시스템 개발 담당 | 크롤링 시스템 리팩토링으로 **처리량 2.5배 확장** (10만→25만 건) |
| **DB 최적화** | MySQL 쿼리 최적화 수행 | Full-Text Index 전환으로 응답 시간 **93% 단축** (3초→0.2초) |
| **도구 개발** | Spring Boot 기반 백엔드 개발 | MCP 서버 설계로 **반복 수작업 주당 5시간 절감** |
| **기술 도입** | Redis Lock 적용 | 동시성 이슈 해결로 **데이터 정합성 100% 확보** |
| **검색 기능** | ElasticSearch 구축 | 검색시간 **1.2초→200ms** 개선, 이탈률 15% 감소 |

## 7년차 Plus Alpha 전략

### 1. 기술적 트레이드오프 & 의사결정 근거

> 시니어는 정답만 맞히는 사람이 아니라 **'선택의 이유'**를 아는 사람

**작성법:**
- ❌ "A 기술을 썼다"
- ✅ "B와 A를 비교했으나, 당시 트래픽 패턴과 리소스 제약을 고려하여 A를 선택"

**의사결정 키워드:**
- API Limit / Rate Limit
- SLA / RPO / RTO
- Failover / 장애 대응
- Infra Cost / 운영 비용

### 2. 팀/조직 영향력 (Engineering Culture)

> 7년차는 혼자 잘하는 것을 넘어 **팀 전체의 생산성을 높여야**

**포함할 경험:**
- 코드 리뷰 문화 정착
- 신규 입사자 온보딩 문서화
- 사내 기술 세미나 주최
- 공통 모듈 개발
- 주니어 멘토링

### 3. 비즈니스 언어 소통

> 기술 용어를 모르는 동료에게 **어떻게 설명하고 설득했는지**

**작성 포인트:**
- 운영팀의 Pain Point를 기술로 어떻게 해결했는지
- PM, 기획자와의 협업 과정

## 작성 예시

### Professional Summary (핵심 요약)

> 채용 담당자가 10초 안에 핵심 역량을 파악할 수 있도록 작성

```
"비즈니스 임팩트를 숫자로 증명하는 데이터 드리븐 개발자"
7년간 커머스 도메인에서 대용량 트래픽 처리와 시스템 자동화에 주력.
Java/Spring Boot 기반 MSA 전환을 주도하여 시스템 안정성 확보,
Elasticsearch 도입으로 검색 성능 20배 개선.
```

### 경력 기술 예시 (STAR 적용)

**성능 개선:**
- 문제: 레거시 시스템의 검색 속도 저하로 사용자 이탈 증가
- 해결: RDB → Elasticsearch 전환, N-gram 인덱싱 및 캐싱 적용
- 성과: 검색 응답 **2초 → 0.1초 (20배)**, 트래픽 30% 증가 수용

**자동화:**
- 문제: 마케팅팀 수동 데이터 취합 일 3시간 소요
- 해결: Spring Batch + 외부 API 연동 파이프라인 구축
- 성과: 리포팅 **100% 자동화**, 월 60시간 절감

**안정성:**
- 문제: 트래픽 폭주 시 DB 커넥션 풀 고갈 장애
- 해결: Replication + Redis 캐싱 + Circuit Breaker
- 성과: 동시 접속 **10만 명 무중단** 운영

### 기술 스택 분류

```
Languages: Java (Expert), Python, SQL
Frameworks: Spring Boot, Spring Batch, JPA/Hibernate
Database: MySQL, Redis, Elasticsearch
Infra: AWS (EC2, S3, RDS), Docker, Jenkins
```

## 체크리스트

작성 후 각 경력 항목마다 확인:

- [ ] 문제/배경(Situation)이 명확한가?
- [ ] 기술 선택 이유(Why)가 있는가?
- [ ] 정량적 성과(숫자)가 있는가?
- [ ] "담당했습니다"류 표현을 제거했는가?
- [ ] 비즈니스 임팩트와 연결되는가?
- [ ] 트레이드오프/의사결정 근거가 드러나는가?
- [ ] 팀/조직 기여가 포함되어 있는가?
