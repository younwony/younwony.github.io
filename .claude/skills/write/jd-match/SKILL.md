# JD 맞춤형 지원서 작성

> **원본 데이터**: `docs/career/my_career_data.md`
> **출력 위치**: `docs/career/formats/by-jd/{company}/`

---

## 개요

JD(채용공고)를 분석하여 맞춤형 지원서를 작성합니다.

### 출력 파일

```
docs/career/formats/by-jd/{company}/
├── JD_ANALYSIS.md      # JD 분석 및 매칭 포인트
├── COVER_LETTER.md     # 자기소개서 (기업 가치 기반)
└── RESUME_JD.md        # JD 맞춤형 이력서
```

---

## 실행 순서

### Step 1: JD 분석

```markdown
**분석 항목:**
1. 필수 요건 추출
2. 우대 사항 추출
3. Pain Point 추론 (회사가 겪고 있을 기술적 과제)
4. 기업 가치/문화 확인 (자기소개서 질문용)
```

### Step 2: 경험 매칭

```markdown
**매칭 기준:**
1. 필수 요건 충족 여부
2. 우대 사항 강점 파악
3. Pain Point에 대한 솔루션 경험
4. 기업 가치와 연결되는 경험 스토리
```

### Step 3: 파일 생성

1. **JD_ANALYSIS.md**: JD 분석 및 매칭 포인트 정리
2. **COVER_LETTER.md**: 기업 가치 기반 자기소개서
3. **RESUME_JD.md**: JD 키워드 강조 이력서

---

## JD 분석 프레임워크

### 1. 필수 요건 매칭 테이블

```markdown
| 요건 | 매칭 | 내 경험 |
|------|------|---------|
| Java 5년 이상 | 충족/부분/부족 | 구체적 경험 |
```

### 2. Pain Point 추론

> 단순 키워드 매칭이 아닌, **회사가 겪고 있는 기술적 통증** 추론

```markdown
**JD 키워드 → Pain Point 추론:**
- "대규모 트래픽" → 현재 성능 이슈 또는 확장 계획
- "MSA 전환" → 레거시 모놀리식 시스템 존재
- "글로벌 확장" → 다국어/다지역 지원 필요
```

### 3. 기업 가치 매칭 (자기소개서용)

```markdown
**기업 가치 → 내 경험 매칭:**
- Transform (변화 주도) → ChatOps 도입, TDD 문화 정착
- Innovation (혁신) → AI 도구 활용, 새로운 아키텍처 도입
- Collaboration (협업) → Cross-functional 리딩, 팀 생산성 향상
```

## 자기소개서 작성 가이드

### 구조 (STAR 기반)

```markdown
**[상황]**: 당시 조직의 문제/과제
**[행동]**: 내가 주도한 해결 방법
**[결과]**: 정량적 성과 + 조직 변화
**[연결]**: 기업 가치와의 연결
```

### 톤

- 구체적인 숫자와 성과 포함
- "~했습니다" 보다 "~달성했습니다" 능동적 표현
- 기술 나열보다 **문제 → 해결 → 성과** 구조

### 분량

- 메인 스토리: 600~800자
- 추가 어필 포인트: 선택적

## JD 맞춤형 이력서 가이드

### 구조 조정

```markdown
**기본 순서**: Summary → Skills → Experience
**JD 맞춤**: Summary에 JD 키워드 강조, 관련 프로젝트 최상단 배치
```

### Professional Summary 작성

```markdown
**공식:**
[연차/직무] + [JD 핵심 키워드] + [가장 큰 성과] + [지원 동기 암시]

**예시:**
"Java/Spring 기반 8년차 백엔드 개발자입니다.
**대규모 트래픽 처리와 Cloud Native 전환** 경험을 바탕으로..."
```

### 프로젝트 배치 전략

```markdown
**JD 우선순위에 따른 배치:**
1. JD 필수 요건과 직접 매칭되는 프로젝트
2. 우대 사항과 매칭되는 프로젝트
3. 기타 주요 성과
```
