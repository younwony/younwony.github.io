# 이력서 작성 가이드 (Skill)

> 이 스킬은 이력서/경력기술서 작성 시 자동으로 참조됩니다.

---

## STAR+I 기법

모든 프로젝트 설명은 STAR+I 형식을 따릅니다:

| 항목 | 설명 | 핵심 질문 |
|------|------|----------|
| **S** (Situation) | 배경·문제·제약 | 어떤 어려움이나 비효율이 있었는가? |
| **T** (Task) | 과제/목표 | 해결해야 할 과제는 무엇인가? |
| **A** (Action) | 기술적 의사결정 (Why & How) | **왜 그 기술을 선택했으며, 어떻게 설계했는가?** |
| **R** (Result) | 정량적 성과 | 수치로 증명된 결과는 무엇인가? |
| **I** (Impact) | 비즈니스/조직적 효과 | 어떤 비즈니스 임팩트를 만들었는가? |

### 문장 템플릿

```
"[제약/문제](S) 속에서 [기술적 의사결정](A)으로 [정량적 성과](R) 달성,
[비즈니스/운영 효과](I)"
```

### 예시

> "API Limit 25k 제약(S) 속에서 우선순위 큐 기반 가격조정 로직 설계(A)로
> 핵심 상품 대응률 90% 달성(R), 운영팀 수작업 완전 제거로 리소스 집중도 향상(I)"

## Bad vs Good 표현

| 구분 | Bad (단순 나열) | Good (문제 해결 & 성과 중심) |
|------|----------------|------------------------------|
| **데이터 수집** | TikTok 데이터 크롤링 시스템 개발 담당 | 단일 책임 원칙 적용으로 유지보수성 향상, **일일 처리량 2.5배 확장** |
| **DB 최적화** | MySQL 쿼리 최적화 수행 | Full-Text Index(ngram) 전환으로 **응답 시간 93% 단축** (3초→0.2초) |
| **도구 개발** | Spring Boot 기반 백엔드 개발 | MCP 서버 설계로 자동화 구현, **주당 5시간 절감** |
| **기술 도입** | Redis Lock 적용 | Redis Distributed Lock으로 **데이터 정합성 100% 확보** |

## 정량화 방식

**수치가 없으면 추정치라도 넣으세요.**

| 분야 | 예시 수치 |
|------|----------|
| 처리량 | 10만건 → 25만건 (2.5배) |
| 검색시간 | 2초 → 200ms (90% 단축) |
| 비용 | 80% 절감 |
| 운영시간 | 100% 제거 (Zero Ops) |
| 정확도 | 85% → 97% |
| 장애율 | 0건 달성 |

## 7년차 Plus Alpha 전략

### 1. 기술적 트레이드오프와 의사결정 근거

> 시니어는 정답만 맞히는 사람이 아니라, **'선택의 이유'**를 아는 사람입니다.

**작성 방법**:
- "A 기술을 썼다"가 아니라
- **"B와 A를 비교했으나, 트래픽 패턴과 리소스 제약을 고려하여 A를 선택했다"**

**의사결정 근거 키워드**:
- API Limit / Rate Limit
- SLA / RPO / RTO
- Failover / 장애 대응
- Infra Cost / 운영 비용

### 2. 팀과 조직에 미친 영향력

**포함해야 할 경험**:
- 코드 리뷰 문화 정착
- 신규 입사자 온보딩 문서화
- 공통 모듈 개발로 팀 리소스 절감
- 주니어 멘토링

### 3. 비즈니스 언어로 소통하는 능력

**작성 포인트**:
- 기술 용어를 모르는 동료에게 어떻게 설명하고 설득했는지
- 운영팀의 Pain Point를 기술로 어떻게 덜어주었는지

## 작성 체크리스트

각 경력 항목마다 확인:

- [ ] 문제/배경이 명확한가?
- [ ] 기술 선택 이유(Why)가 있는가?
- [ ] 정량적 성과(숫자)가 있는가?
- [ ] "담당했습니다"류 표현을 제거했는가?
- [ ] 비즈니스 임팩트와 연결되는가?
- [ ] 트레이드오프/의사결정 근거가 드러나는가?
