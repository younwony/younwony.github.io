# 이력서 + 경력기술서 세트 생성

## 개요

채용 지원 시 보통 **이력서**와 **경력기술서** 두 가지를 함께 요청합니다.
이 스킬은 두 문서를 일관된 스타일로 함께 생성합니다.

## 문서 구분

| 구분 | 이력서 (Resume) | 경력기술서 (Career Portfolio) |
|------|----------------|------------------------------|
| **분량** | 1-2페이지 | 3-5페이지 이상 |
| **목적** | 빠른 스크리닝 | 상세 역량 검증 |
| **내용** | 핵심 요약, 성과 중심 | 프로젝트 상세, STAR+I |
| **톤** | 간결, 임팩트 | 구체적, 서술적 |

## 출력 파일

```
templates/export/pdf/
├── resume-2page.html           # 이력서 (2-3페이지)
└── career-portfolio.html       # 경력기술서 (상세)
```

## 이력서 구성 (2-3페이지)

### 2-3페이지 이력서 구조

```
[페이지 1]
┌─────────────────────────────────────┐
│ 헤더: 이름, 직함, 연락처             │
├─────────────────────────────────────┤
│ Key Metrics (핵심 성과 4개)          │
├─────────────────────────────────────┤
│ Technical Skills (카테고리별)        │
├─────────────────────────────────────┤
│ Work Experience (시작)              │
└─────────────────────────────────────┘

[페이지 2]
┌─────────────────────────────────────┐
│ Work Experience (계속)              │
├─────────────────────────────────────┤
│ Key Projects                        │
│ - 프로젝트 1-4                       │
└─────────────────────────────────────┘

[페이지 3] (선택)
┌─────────────────────────────────────┐
│ Key Projects (continued)            │
├─────────────────────────────────────┤
│ Education & Certification           │
├─────────────────────────────────────┤
│ Awards                              │
└─────────────────────────────────────┘
```

### 이력서 작성 원칙

1. **숫자로 말하라**: 모든 성과는 정량화
2. **핵심만 남겨라**: 3줄 이상 쓰지 않기
3. **최신 경력 우선**: 최근 경력에 분량 집중
4. **키워드 배치**: JD 필수 기술은 Bold 처리

### 이력서 포함 내용

- ✅ 핵심 성과 (숫자)
- ✅ 기술 스택 요약
- ✅ 최근 3개 회사
- ✅ 학력/자격증
- ❌ 프로젝트 상세 설명
- ❌ 기술 선택 이유
- ❌ 비즈니스 임팩트 상세

## 경력기술서 구성 (3-5페이지)

### 경력기술서 구조

```
┌─────────────────────────────────────┐
│ 페이지 1: 헤더 + Summary + Skills    │
├─────────────────────────────────────┤
│ 페이지 2-4: 프로젝트 상세 (STAR+I)   │
│   - 프로젝트 1 (최근/주요)           │
│   - 프로젝트 2                       │
│   - 프로젝트 3                       │
│   - 프로젝트 4                       │
│   - 프로젝트 5                       │
├─────────────────────────────────────┤
│ 페이지 5: 이전 경력 + 학력           │
└─────────────────────────────────────┘
```

### 경력기술서 작성 원칙

1. **STAR+I 형식 필수**
   - Situation: 배경/문제점
   - Task: 과제/목표
   - Action: 해결 방법 (기술 선택 이유 포함)
   - Result: 정량적 성과
   - Impact: 비즈니스 효과

2. **기술 의사결정 강조**
   - "왜 이 기술을 선택했는가?"
   - "어떤 트레이드오프가 있었는가?"

3. **비즈니스 임팩트 연결**
   - 기술 성과 → 비즈니스 효과

### 경력기술서 포함 내용

- ✅ 프로젝트 상세 (STAR+I)
- ✅ 기술 선택 이유 (Tech Decision)
- ✅ 비즈니스 임팩트
- ✅ 협업/리더십 경험
- ✅ 사용 기술 상세

## 빠른 생성 방법

### PDF 내보내기

> **중요**: PDF 내보내기는 `/export-pdf` 스킬을 참조하세요.
> 페이지 경계에서 내용이 끊기지 않도록 여백과 페이지 나눔을 최적화합니다.

```bash
# 1. HTML 열기
start templates/export/pdf/resume-2page.html      # 이력서 (2-3페이지)
start templates/export/pdf/career-portfolio.html  # 경력기술서 (다중 페이지)

# 2. Ctrl+P → "PDF로 저장" 선택
#    - 여백: 없음 또는 최소
#    - 배경 그래픽: 체크
```

### PPT/DOCX 내보내기

> PPT/DOCX 내보내기는 `/export-document` 스킬을 참조하세요.

```bash
node scripts/generate-documents.js

# 결과: output/
# - 윤원희_자기소개.pptx
# - 윤원희_경력기술서.pptx
```

## 실행 단계

### Step 1: 원본 데이터 확인

```bash
# 최신 데이터 확인
git diff docs/career/my_career_data.md
```

### Step 2: 이력서 생성

1. `resume-2page.html` 템플릿 사용
2. 핵심 성과 중심으로 간소화
3. 2-3페이지에 딱 맞게 조정
4. 브라우저 미리보기 → PDF 저장

### Step 3: 경력기술서 생성

1. `career-portfolio.html` 사용
2. 모든 프로젝트 STAR+I 형식 확인
3. 페이지 경계에서 내용 잘림 확인
4. 브라우저 미리보기 → PDF 저장

### Step 4: 일관성 검토

- [ ] 두 문서의 연락처/학력 정보 일치
- [ ] 성과 수치 일치
- [ ] 기술 스택 표기 일치
- [ ] 회사명/기간 표기 일치

## 예시: 동일 프로젝트의 두 버전

### 이력서 버전 (2-3줄)

```
인플루언서 데이터 플랫폼 (200만 건)
- 데이터 풀 20배 확대 (10만→200만), 검색 성능 100배 개선 (10초→0.1초)
- Elasticsearch, Redis Lock 기반 대용량 처리 아키텍처 구축
```

### 경력기술서 버전 (STAR+I)

```
프로젝트: 인플루언서 데이터 플랫폼 고도화 (200만 데이터 규모)

📌 Situation (배경)
- 마케팅팀 수작업 의존: 엑셀 수기 입력, 일 수십 명 수준
- 검색 성능 저하: 200만 건 복합 조건 검색 시 10초 이상 타임아웃

🎯 Task (과제)
- 수집 프로세스 100% 자동화
- 200만 건 규모 실시간 검색 시스템 구축

⚡ Action (해결)
- 멀티소스 데이터 파이프라인: TikTok API, Ensemble API 조합, 병렬 처리
- Elasticsearch 도입: RDB 대비 검색 성능 100배 개선
- Redis Distributed Lock: 멀티 인스턴스 환경 중복 수집 방지

✅ Result (성과)
- 수집 자동화 100%, 데이터 풀 20배 확대 (10만→200만)
- 검색 성능 100배 개선 (10초→0.1초)

💡 Impact (비즈니스 효과)
- 대형 캠페인 진행 가능한 마케팅 커버리지 확보
- 리포팅 업무 제거로 마케팅팀 고부가가치 업무 집중

🔧 Tech Decision
> "RDB 대신 Elasticsearch 선택 - 역색인 기반으로 복합 조건 검색 0.1초 달성"
> "Redis Lock 도입 - API Rate Limit 준수 및 중복 수집 방지"
```
