## 3. 경력 기술 (Work Experience) - STAR 기법 적용
경력직 이력서의 핵심은 **'무엇을 했는지(Task)'**보다 **'어떤 성과를 냈는지(Result)'**입니다.

Task(과제) 단계에서 기술적 난이도나 비즈니스 목표가 구체적으로 드러나야 합니다.

### 
**[작성 패턴: 배경(Situation) -> 과제(Task) -> 행동(Action) -> 성과(Result)]**

S (Situation - 배경): 프로젝트 당시의 상황, 문제점, 레거시 시스템의 한계.

T (Task - 과제/목표): 해결해야 했던 구체적인 문제, 달성해야 할 목표 수치, 기술적 제약 사항.

A (Action - 행동/해결): 본인이 수행한 핵심 역할, 기술적 의사결정(Why), 적용한 아키텍처 및 알고리즘.

R (Result - 성과): 정량적 수치(%), 비즈니스 임팩트, 팀 내 변화.

### **Case 1: 대용량 데이터 처리 (인플루언서 플랫폼)**
**순히 "많이 수집했다"가 아니라, "어떤 제약 속에서 목표를 달성했는지" 보여주는 예시**

- Situation (배경): * 기존 단일 스레드 기반 수집기로는 시간당 처리량이 100건에 불과해 캠페인 확장이 불가능했음.
  - 외부 API(TikTok 등)의 엄격한 Rate Limit으로 인해 무작정 요청을 늘릴 수 없는 상황.
- Task (과제): * API 차단 없이 데이터 수집 속도를 기존 대비 100배(10,000건/h) 이상으로 끌어올려야 함.
  - CDN 링크 만료로 인한 이미지 유실을 0%로 만들어 데이터 정합성을 확보해야 함.
- Action (해결): * 병렬 아키텍처 설계: ThreadPoolTaskExecutor를 커스텀하여 I/O Blocking 시간을 최소화하고 리소스 효율 극대화.
  - 안정성 확보: Resilience4j RateLimiter와 Backoff Retry 정책을 적용해 API 임계치 내에서 최대 효율을 내도록 튜닝.
  - 분산 제어: 멀티 인스턴스 환경에서 중복 수집을 막기 위해 Redis Distributed Lock 도입.
- Result (성과): * 수집 속도 99% 단축 (100명/h → 10,000명/h).
  - 데이터 풀 20배 확대 (10만 → 200만 명)로 마케팅 커버리지 극대화.

#### **Case 2: 성능 최적화 (동적 가격 시스템)**
**RDB의 한계를 극복하고 검색 엔진을 도입한 논리적 흐름 강조**

- Situation (배경): * 수백만 건의 상품 데이터에서 경쟁사 최저가를 실시간으로 비교해야 했으나, RDB 쿼리 성능 저하로 업데이트가 6시간 이상 지연됨.
- Task (과제): * 가격 비교 조회 속도를 ms 단위로 단축하여 실시간 대응 체계 구축.
  - 대규모 배치 작업 중에도 DB 부하를 최소화하여 서비스 안정성 유지.
- Action (해결): * 저장소 전환: 복합 조건 검색(브랜드+모델)에 유리한 Elasticsearch를 도입하여 역색인 구조 활용.
  - 배치 최적화: Spring Batch의 Chunk 사이즈를 튜닝하고, 병렬 스텝(Parallel Step)을 적용해 대량 트랜잭션 처리 시간을 단축.
- Result (성과): * 가격 대응 속도 실시간(Real-time) 수준으로 개선.
  - 가격 경쟁력 확보를 통해 매출 10% 증대 및 유입 트래픽 15% 상승.

#### **Case 3: 운영 자동화 (ChatOps)**
**비효율을 제거하고 시스템화한 리더십 강조**

- Situation (배경): * 파트너사의 잦은 품절과 수동 발주 처리로 인해 운영팀 업무의 40%가 단순 반복 작업에 소모됨.
  - 데이터가 아닌 '감'에 의존하여 클레임 원인 파악이 불가능했음.
- Task (과제): * 운영팀의 개입이 전혀 없는 'Zero Ops' 환경 구축.
  - 로그 데이터를 기반으로 품절 리스크를 사전에 탐지하는 모니터링 체계 마련.
- Action (해결): * 데이터 파이프라인: GA(행동)와 DB(주문) 로그를 BigQuery로 통합 적재하여 '상습 품절 패턴' 분석 쿼리 작성.
  - 프로세스 통합: Slack Event API를 활용해 메신저 내에서 '승인/반려' 버튼 클릭만으로 업무가 종결되도록 ChatOps 구현.
- Result (성과): * 전체 품절률 20% 감소 및 클레임 사전 차단.
  - 운영팀 반복 업무 100% 제거로 고부가가치 업무 집중 환경 마련.

## 5. 기타 팁
- ***수치화:** "성능 개선"보다는 "**응답 속도 50% 단축**"과 같이 구체적인 숫자(정략적 수치)를 사용하세요.
  - 나쁜 예: "성능을 개선해야 했음"
  - 좋은 예: "응답 속도를 2초에서 0.5초 이내로 줄이고, 동시 접속자 1만 명을 버텨야 했음"
- ***Action에는 단순히 사용한 '기술명'만 나열하지 말고 해당 기술을 선택한 '의도'를 적으세요:***
  - 나쁜 예: "Redis를 사용함"
  - 좋은 예: "DB 부하를 줄이고 조회 속도를 높이기 위해 **Redis 캐싱 레이어(Look-aside 패턴)**를 적용함"
- ***키워드 매칭:** 채용 공고(JD)에 있는 핵심 키워드(예: MSA, AWS, 대용량 트래픽)를 이력서 곳곳에 배치하세요.
- ***가독성:** 불렛 포인트(Bullet points)를 적극 활용하고, 핵심 성과는 굵은 글씨로 강조하세요.
