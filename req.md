# 경력기술서 작성 명세서 v5.0

> **핵심**: 한 문장에 **[해결 방법] + [정량 성과]**를 자연스럽게 녹여서 작성

---

## 1. 핵심 원칙

1. **성과 중심 서술**: 각 bullet에 문제→해결→결과가 한 문장으로
2. **정량 지표 필수**: 모든 성과에 숫자 포함
3. **상위 20% 프로젝트**만 선별 (5~6개)

---

## 2. 프로젝트 구조

```markdown
### 프로젝트명 (YYYY.MM - YYYY.MM)

**역할**: 백엔드 리드 (기여도 70%, 4인 팀 리딩)

**Tech Stack**: Java 17, Spring Boot 3.0, Redis, MySQL, AWS

**[주요 성과]**
- 성과 제목 (Before → After): 해결 방법을 적용하여 문제를 해결하고 결과를 달성.
- 성과 제목 (수치): 기술적 접근 방식으로 문제를 해결.
- 성과 제목: 구체적인 해결 방법과 비즈니스 임팩트.
```

---

## 3. 성과 문장 작성법

### 3.1 기본 패턴

```
[성과 제목] (정량 지표): [해결 방법]을 적용하여 [문제]를 해결.
```

### 3.2 작성 예시

```markdown
❌ 나쁜 예시 (방법만 나열)
- Elasticsearch를 도입하여 검색 기능을 구현함
- Redis를 사용하여 캐싱 처리함

✅ 좋은 예시 (성과 중심)
- 검색 성능 100배 향상 (10초 → 0.1초): Elasticsearch 역색인 기반 검색 엔진을 구축하여
  200만 건 복합 조건 검색 시 RDB 타임아웃 문제 해결.
- 데이터 수집 자동화로 마케팅 커버리지 20배 확대: 멀티소스 API 파이프라인과
  Redis 분산 락을 적용하여 수작업 의존도를 100% 제거.
```

### 3.3 성과 제목 유형

| 유형 | 예시 |
|------|------|
| **성능 개선** | 검색 성능 100배 향상 (10초 → 0.1초) |
| **비용 절감** | API 호출량 80% 절감 (10,000건 → 2,000건/일) |
| **자동화** | 운영 업무 Zero Ops 달성 (수동 40% → 0%) |
| **안정성** | 클레임 Zero 달성 (월 20건+ → 0건) |
| **매출 기여** | 대상 상품 매출 10% 상승 |

## 4. 실전 예시

```markdown
### 인플루언서 데이터 플랫폼 고도화 (2022.02 - 현재)

**역할**: 백엔드 아키텍처 설계 주도 (1인 개발)

**Tech Stack**: Spring Boot, Elasticsearch, Redis Lock, Google Sheets API

**[주요 성과]**
- 검색 성능 100배 향상 (10초 → 0.1초): Elasticsearch 역색인 기반 검색 엔진을 구축하여
  200만 건 복합 조건 검색 시 RDB 타임아웃 문제 해결.
- 데이터 수집 자동화로 마케팅 커버리지 20배 확대 (10만 → 200만): 멀티소스 API 파이프라인과
  Redis 분산 락을 적용하여 수작업 의존(일 수십 명)을 완전 자동화.
- 리포팅 업무 Zero Ops 달성: Google Sheets API 연동으로 마케팅팀 주간 리포트
  작성 업무(반나절/주)를 완전 자동화.
```

## 5. 체크리스트

### 성과 문장별
- [ ] **정량 지표** 포함 (Before → After 또는 수치)
- [ ] **해결 방법**이 구체적으로 명시
- [ ] **한 문장**으로 문제→해결→결과 흐름

### 프로젝트별
- [ ] 역할과 **기여도** 명시
- [ ] Tech Stack 나열
- [ ] **3~4개** 핵심 성과

### 전체 문서
- [ ] 프로젝트 **5~6개** 이내
- [ ] 스캔하기 쉬운 구조
- [ ] 면접 확장 설명 가능한 깊이

---

## 6. 역할 표현 가이드

```markdown
**역할**: 백엔드 리드 (기여도 70%, 4인 팀 리딩)
**역할**: 가격 정책 로직 설계 주도 (기획/운영팀 협업)
**역할**: 검색 엔진 아키텍처 설계 및 구현 (1인 개발)
```

| 요소 | 예시 |
|------|------|
| **역할명** | 백엔드 리드, 설계 주도, 아키텍처 설계 |
| **기여도** | 기여도 70%, 1인 개발, 전 과정 리딩 |
| **팀 구성** | 4인 팀 리딩, 2인 협업 |
| **협업 범위** | 기획팀 협업, 운영팀 협의 |
